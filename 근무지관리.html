근무지 관리 화면은 출퇴근 허용 기준을 운영하는 목록 화면입니다

근무지 관리는 단순 주소 목록이 아니라, 회사가 어떤 장소를 공식 근무지로 운영하고 있는지, 어떤 근무지가 신규 직원 배포 기준인지, 어떤 근무지를 삭제하거나 미사용 처리할 수 있는지를 판단하는 관리 화면입니다. 실제 세부 값은 설정 화면에서 입력하더라도, 운영 상태를 점검하고 정리하는 중심 화면은 이 목록 화면입니다.

화면 개요

이 화면은 등록된 근무지를 한눈에 모아 보고, 현재 운영 중인 기준 근무지가 적절한지 점검하는 화면입니다. 특히 대표 근무지, 사용 인원, 사용 여부가 함께 보이기 때문에, 운영자는 이 화면에서 단순 조회가 아니라 유지할지, 바꿀지, 정리할지를 판단하게 됩니다.

관리 허브 역할

근무지 등록 자체는 설정 화면으로 이어지지만, 전체 현황 파악과 상태 점검은 이 화면에서 이뤄집니다. 그래서 근무지 관리 화면은 근무지 운영의 시작점이자 통제 화면입니다.

직원 적용과의 연결

이곳에서 보이는 대표 근무지와 사용 여부는 신규 직원 기본 배포와 기존 직원 유지 정책으로 이어집니다. 근무지 정보는 직원 출퇴근 인정 방식의 기준이기도 합니다.

목록 화면은 결과 화면이 아니라 운영 화면입니다. 어떤 근무지를 계속 사용할지, 어떤 근무지를 더 이상 신규 배포하지 않을지 판단하는 역할이 가장 중요합니다.

업무 흐름 요약

목록 진입

기본 조건으로 자동 조회해 현재 등록 현황을 확인합니다.

검색 / 점검

명칭 또는 주소 기준으로 필요한 근무지를 찾아 상태를 점검합니다.

등록 / 수정 이동

신규 근무지는 등록으로, 기존 근무지는 상세 수정으로 이어집니다.

삭제 / 미사용 판단

사용 인원과 대표 여부를 기준으로 정리 가능성을 판단합니다.

직원 영향 확인

신규 배포와 기존 직원 유지에 영향이 없는지 최종 확인합니다.

검색 영역

근무지 목록은 화면 진입 시 기본 조건으로 자동 조회됩니다. 별도 검색어를 넣지 않아도 현재 운영 중인 근무지 현황을 바로 확인할 수 있어야 하기 때문입니다. 검색은 특정 근무지를 찾는 기능이면서, 중복 근무지 여부를 점검하는 수단이기도 합니다.

항목기준설명
최초 진입기본값 자동 조회화면이 열리면 현재 목록이 자동으로 조회되어 빈 화면 없이 바로 점검할 수 있습니다.
검색 구분전체 / 명칭 / 주소검색 드롭다운을 통해 어떤 기준으로 찾을지 선택합니다.

결과 없음안내 문구 노출조회 결과가 없을 때는 결과 없음 메시지를 표시해 조건 불일치 상황을 분명히 보여줍니다.
주소 기준 검색은 같은 장소가 다른 이름으로 중복 등록되는 것을 줄이는 데 특히 유용합니다.

상단 버튼과 일괄 작업

상단 버튼은 근무지 운영에 필요한 주요 작업을 빠르게 실행하는 장치입니다. 다만 버튼을 누르기 전에, 사용 인원과 대표 근무지 여부를 먼저 확인하는 것이 운영상 더 중요합니다.

등록

새로운 사업장, 지점, 현장을 근무지로 추가할 때 사용합니다. 실제 세부 입력은 설정 화면에서 진행합니다.

수정

기존 근무지의 주소, 반경, 체크 방식, 대표 여부를 바꿔야 할 때 사용합니다.

삭제

더 이상 운영하지 않는 근무지를 제거할 때 사용하지만, 사용 인원과 대표 여부에 따라 제한될 수 있습니다.

복사

유사한 설정을 가진 근무지를 빠르게 추가할 때 사용합니다. 여러 지점이 비슷한 기준을 공유할 때 유용합니다.

출력

현황 점검 자료나 운영 회의 자료로 활용할 수 있도록 현재 조회 결과를 출력합니다.

엑셀 다운로드 / 업로드

초기 구축, 대량 정비, 대규모 점검에 사용하는 기능입니다. 업로드 전에는 대표 근무지와 사용 여부 기준을 먼저 정리해야 합니다.

상단에는 전체 건수와 선택 건수가 함께 표시됩니다. 전체 건수는 운영 범위를, 선택 건수는 일괄 작업 대상 수를 보여주므로 삭제나 복사 전 확인이 필요합니다.

대표 근무지 / 사용 여부 운영 규칙

대표 근무지

대표 근무지는 신규 직원에게 기본으로 연결되는 근무지입니다. 따라서 대표 근무지는 단순 표시값이 아니라 조직의 기본 출퇴근 기준을 의미합니다.

  • 대표 근무지는 목록 최상단에 배치됩니다.
  • 대표 근무지는 삭제 대상에서 제외됩니다.
  • 대표 근무지가 바뀌면 신규 배포 기준도 함께 바뀝니다.

사용 여부

사용 여부는 과거 이력을 지우는 기능이 아니라, 앞으로 신규 배포 기준으로 계속 쓸 것인지를 정하는 상태값입니다.

사용

신규 직원 배포 대상에 포함되며 현재 운영 중인 근무지로 관리됩니다.

미사용

신규 배포에서는 제외되지만 기존 직원과 과거 이력은 유지됩니다. 삭제 전에 단계적으로 정리할 때 적합합니다.

사용 인원이 있는 근무지를 바로 삭제하기보다, 먼저 미사용으로 전환하고 배포 기준을 다른 근무지로 옮긴 뒤 정리하는 것이 더 안전한 운영 방식입니다.

삭제 / 미사용 / 일부삭제 컨펌과 얼럿

목록 화면의 메시지는 단순 경고가 아니라 운영 기준을 지키게 하는 정책 장치입니다. 특히 사용 인원 존재 여부, 대표 근무지 여부, 일괄 삭제 가능 범위에 따라 서로 다른 안내가 표시됩니다.

안내사용 중인 직원이 있는 근무지는 삭제할 수 없습니다.

운영 의미: 이미 직원에게 배포된 근무지가 갑자기 사라지지 않도록 막아 출퇴근 인정 기준의 혼선을 방지합니다.

안내사용 인원이 있는 근무지를 미사용으로 변경하려면 영향 범위를 확인해야 합니다.

운영 의미: 미사용 전환은 가능하더라도, 현재 배포된 직원이 있는지 먼저 점검해야 함을 안내합니다.

안내대표 근무지는 삭제할 수 없습니다.

운영 의미: 대표 근무지는 신규 직원 기본 배포 기준이므로 별도 대표 변경 없이 제거되지 않도록 보호합니다.

확인삭제 가능한 근무지 N건을 삭제하시겠습니까.

운영 의미: 여러 건 삭제 시 실제 처리 건수를 다시 확인하게 해 일괄 삭제 실수를 줄입니다.

확인선택 건 중 삭제 가능한 대상만 삭제합니다.

운영 의미: 일부만 삭제 가능한 상황을 분리해 보여주어, 왜 모든 선택 건이 처리되지 않았는지 이해할 수 있게 합니다.

운영자가 이 화면에서 최종 확인해야 할 것

대표 근무지가 적절한가

신규 직원 기본 배포 기준으로 사용할 근무지가 맞는지 확인합니다.

미사용 전환 대상이 맞는가

더 이상 신규 배포가 필요 없는 근무지인지, 아직 운영 중인지를 구분합니다.

사용 인원이 남아 있는가

삭제 또는 미사용 전환 전에 현재 배포된 직원 수를 반드시 확인합니다.

중복 근무지가 없는가

명칭과 주소 기준으로 유사한 근무지가 여러 번 등록되지 않았는지 점검합니다.
