출장·외근 유형 관리는 타각이 부족한 업무를 간주근무로 보정하는 계산 정책 화면입니다

출장유형 설정 화면은 출장명만 관리하는 사전이 아닙니다. 출퇴근을 인정할지, 근무시간을 자동 인정할지, 몇 시간을 인정할지, 휴게를 몇 시간으로 볼지, 휴일근무와 연장·야간을 같이 인정할지까지 정해서 실제 타각이 없거나 불완전한 업무를 근태 계산에서 어떻게 반영할지를 결정하는 정책 화면입니다.

메뉴: 출장유형 설정 관련 기능: 출장유형 등록 핵심 역할: 간주근무, 인정시간, 휴일/연장/야간 분기
목차
화면이 중요한 이유 기본 구조 등록·수정 시 입력하는 상세 정책 핵심 정책별 계산 해석 정책 조합 예시 운영 시 주의할 점 인정근로시간·월마감과의 연결 운영 체크리스트 같이 보면 좋은 문서

화면이 중요한 이유

타각 대체 기준

출장, 외근, 현장업무처럼 정상적인 출퇴근 타각이 어려운 경우가 있습니다. 이 화면은 그런 케이스를 자동 인정으로 처리할지, 실제 타각이 있어야만 인정할지, 정해진 근무시간을 간주로 넣을지 결정합니다.

가산시간 분해 기준

연장, 야간, 휴일 여부를 이 유형에서 같이 판단하도록 설계되어 있어, 단순 근무시간 인정만이 아니라 수당성 시간 분해 규칙까지 함께 관리하는 화면으로 봐야 합니다.

기본 구조

영역 화면 내용 매뉴얼 해석
검색 영역 출장 유형 목록을 조회하기 위한 기본 검색영역이 존재합니다. 사용 중인 유형, 사용 종료 유형, 특정 출장명 등을 운영자가 빠르게 찾아볼 수 있는 목록형 관리 화면입니다.
리스트 영역 등록된 출장 유형이 표 형태로 노출됩니다. 목록에서 어떤 간주 정책이 현재 운영 중인지 한눈에 확인할 수 있어야 하므로 사용여부, 유형명, 인정규칙이 핵심 노출 값입니다.
등록 화면 출장유형 등록 화면에서 신규 출장 유형을 생성합니다. 정책 변경이 잦을 수 있으므로 목록과 별도 팝업으로 생성·수정하는 구조입니다.
수정/삭제/저장 유형을 수정하거나 삭제하고, 취소와 저장 버튼이 함께 제공됩니다. 간주근무 정책은 과거 근태에 영향을 줄 수 있으므로 취소, 저장, 사용여부 전환 흐름이 분명해야 합니다.

등록·수정 시 입력하는 상세 정책

필드 설정 기준 업무 의미
근태 유형 기본값 고정: 출장 이 화면이 휴가나 일반 근태 수정이 아니라 출장/외근 계열 간주근무 정책 화면임을 명확히 합니다.
출장 구분(출장명) 유형명 입력 직원이 신청서에서 선택하게 될 실사용 명칭입니다. 국내출장, 당일외근, 현장지원, 고객사상주 같은 업무 유형을 정의합니다.
출장 인정 단위 일자 / 기간 하루 단위 출장인지, 시작일과 종료일이 있는 기간형 출장인지 구분합니다. 신청서 UI와 계산 단위가 달라집니다.
출/퇴근 인정 인정 / 미인정 타각이 없더라도 출근과 퇴근으로 간주할지 정합니다. 미인정이면 별도 타각 또는 보정 요청이 필요할 수 있습니다.
근무 인정 시간 시간 + 분 선택 정해진 출장 유형에 대해 몇 시간을 근무로 볼지 결정합니다. 간주근무의 핵심 기준값입니다.
휴게 인정 시간 시간 입력 간주 근무시간 안에 몇 시간을 휴게로 볼지 정합니다. 실제 근무시간과 휴게시간을 분리해 저장하는 구조가 필요합니다.
근무 시간 자동 인정 사용 / 미사용 승인만으로 자동 근무시간을 생성할지, 실제 타각 또는 추가 보정이 있어야 하는지 결정합니다.
휴일 근무 인정 휴일 포함 / 휴일 제외 휴일에 발생한 출장도 휴일근무로 집계할지 여부를 정합니다. 휴일수당 계산과 직접 연결됩니다.
연장/야간 근무 인정 인정 / 미인정 출장 시간 안에서 연장 또는 야간 가산시간을 추가로 분리할 수 있는지 결정합니다.
사용여부 사용 / 미사용 신규 신청 가능 여부를 통제합니다. 과거 이력은 유지하고 신규 사용만 막는 운영 제어용 필드입니다.
비고 설명 입력 정책 해석, 예외 처리 기준, 사용 부서 제한 같은 운영 메모를 남기는 필드입니다.

핵심 정책별 계산 해석

근태 계산과 직접 연결되는 항목
정책 항목 설정값 근태 계산 반영 방식
출/퇴근 인정 인정 실제 타각이 없더라도 출장 승인 데이터만으로 출근/퇴근 구간을 간주 생성할 수 있습니다.
출/퇴근 인정 미인정 출장 승인만으로는 근무구간을 확정하지 못하므로 실제 타각, 수기보정, 별도 근태요청이 필요할 수 있습니다.
근무 시간 자동 인정 사용 신청 승인 후 유형에 정의된 근무 인정 시간과 휴게 인정 시간을 바탕으로 일별 간주근무 구간을 자동 생성합니다.
근무 시간 자동 인정 미사용 출장 상태는 남기되 인정근로시간은 별도 타각 또는 보정결과를 사용합니다.
휴일 근무 인정 휴일 포함 휴일에 적용된 출장도 휴일근무시간으로 집계할 수 있습니다.
휴일 근무 인정 휴일 제외 휴일에 출장 일정이 있어도 휴일근무로는 보지 않거나 별도 수기판정을 요구하는 정책이 됩니다.
연장/야간 근무 인정 인정 간주된 근무구간 안에서도 기준시각을 넘긴 시간은 연장 또는 야간시간으로 다시 분해합니다.
연장/야간 근무 인정 미인정 출장 인정 시간은 총 근무시간에는 포함되더라도 가산시간으로는 분리되지 않습니다.

정책 조합 예시

예시 1. 당일 출장

출/퇴근 인정 = 인정, 근무 시간 자동 인정 = 사용, 근무 인정 시간 = 8시간, 휴게 인정 시간 = 1시간이면 타각이 없어도 8시간 간주근무와 1시간 휴게가 자동 생성됩니다.

예시 2. 현장 외근

출/퇴근 인정 = 미인정, 자동 인정 = 미사용이면 출장 상태만 기록하고 실제 타각 또는 관리자 보정을 별도로 받아야 합니다.

예시 3. 휴일 행사 지원

휴일 근무 인정 = 휴일 포함, 연장/야간 인정 = 인정이면 휴일 출장도 휴일근무시간과 야간시간으로 추가 분해할 수 있습니다.

운영 시 주의할 점

  • 출장 유형은 단순 분류가 아니라 근태 계산 기준이므로, 이름 변경보다 인정 규칙 변경이 훨씬 큰 영향도를 가집니다.
  • 근무 인정 시간과 휴게 인정 시간을 함께 들고 있어야 총 체류시간과 실근로시간을 분리할 수 있습니다.
  • 자동 인정이 켜진 유형은 승인만으로 근무시간이 생성되므로, 승인 완료 시점과 마감 반영 시점의 관계를 명확히 정의해야 합니다.
  • 휴일 포함 여부와 연장/야간 인정 여부는 총 인정시간과 가산시간을 분리 저장해야 한다는 신호입니다.
  • 기간 단위 출장 유형은 시작일과 종료일 사이의 일자별 분해 규칙도 함께 필요합니다.

인정근로시간·월마감과의 연결

일별 판정 단계

출장 신청이 승인되면 시스템은 먼저 출장유형 기준정보를 읽고, 자동 인정 여부에 따라 예상 근무 프레임을 대체하거나 실제 타각과 결합합니다. 이때 근무 인정 시간과 휴게 인정 시간이 함께 적용됩니다.

가산시간 분해 단계

휴일 포함 여부와 연장/야간 인정 여부를 확인해서 총 인정근로시간과 별도로 휴일근무시간, 연장근무시간, 야간근무시간을 분해합니다.

마감 결과 항목 출장유형 설정의 영향
인정근로시간 근무 인정 시간, 자동 인정 여부, 출퇴근 인정 여부가 직접 영향을 줍니다.
휴게시간 휴게 인정 시간이 별도 저장되어야 실근로시간과 구분할 수 있습니다.
휴일근무시간 휴일 근무 인정이 켜져 있을 때만 휴일시간으로 분류됩니다.
연장/야간시간 연장·야간 인정이 켜져 있을 때만 가산시간 분해를 허용합니다.
근태이상 여부 자동 인정이 꺼져 있고 타각이 부족하면 이상건으로 남을 수 있습니다.
출장유형은 사실상 간주근무 기준으로 운영됩니다. 따라서 월마감에서도 출장 승인 데이터는 단순 상태 데이터가 아니라 인정근로시간 보정용 입력원천으로 취급하는 것이 맞습니다.

운영 체크리스트

  • 출장 유형을 만들 때는 출장명보다 자동 인정 여부와 인정 시간을 먼저 결정합니다.
  • 휴일 포함 여부와 연장·야간 인정 여부는 급여 반영까지 이어지므로 실제 보상정책과 맞는지 검토합니다.
  • 기간형 출장은 일자별 분해 로직을 별도 문서로 보완합니다.
  • 자동 인정이 켜진 유형은 승인 취소, 변경, 반려 시 이미 생성된 근무시간을 어떻게 되돌릴지 로그 정책을 같이 준비합니다.
