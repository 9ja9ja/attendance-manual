# 근태관리 매뉴얼 — 팀원 온보딩 가이드

## 이 문서의 목적

근태관리 매뉴얼을 AI로 수정하고 자동 배포하는 방법을 안내합니다.
코딩 지식, VS Code, 터미널 명령어는 필요하지 않습니다.

---

## 전체 프로세스 (3단계)

```
1. Claude에게 수정 요청  →  2. 수정 결과 확인  →  3. 배포 (자동)
```

AI에게 "어디를 어떻게 수정해줘"라고 말하면, AI가 파일 수정부터 배포까지 처리합니다.

---

## 1. 필요한 것

| 항목 | 설명 | 비용 |
|------|------|------|
| **GitHub 계정** | 매뉴얼 저장소 접근용 | 무료 |
| **Claude 계정** | AI 수정 요청용 | Pro 구독 |

이 두 개만 있으면 됩니다.

---

## 2. 최초 세팅 (한 번만)

### 2.1 GitHub 계정 만들기

1. https://github.com 접속
2. Sign up → 이메일, 비밀번호 입력 → 가입 완료
3. **GitHub 아이디를 관리자에게 알려주세요** → Collaborator 초대를 보내드립니다
4. 이메일로 온 초대 수락

### 2.2 Claude Code 웹에서 레포 연결

1. https://claude.ai/code 접속 (Claude Pro 로그인)
2. "Connect to GitHub" → GitHub 계정 연결
3. `9ja9ja/attendance-manual` 레포 선택
4. 연결 완료

이제 Claude에게 수정을 요청할 준비가 됐습니다.

---

## 3. 매뉴얼 수정하기

### 수정 요청 예시

Claude Code 웹에서 자연어로 요청하면 됩니다:

**내용 수정:**
```
근로제관리.html에서 검색 영역 설명에
"근무요일 드롭다운 옵션이 추가됨"이라는 내용을 넣어줘
```

**새 문서 추가:**
```
근태권한관리.html 파일을 새로 만들어줘.
관리자별 접근 권한 설정 방법을 설명하는 매뉴얼이야.
기존 매뉴얼과 동일한 스타일로 만들어줘.
@CLAUDE.md 참고해서
```

**용어집 업데이트:**
```
용어집.html에 "탄력근로제" 용어를 추가해줘.
정의: 2주~3개월 단위로 근로시간을 평균하여 배분하는 제도
```

**오타 수정:**
```
휴가관리.html에서 "유효기긴"을 "유효기간"으로 수정해줘
```

### Claude가 수정 후 자동으로 하는 일

1. 파일 수정
2. `git commit` (변경 내역 저장)
3. `git push` (GitHub에 업로드)
4. → Vercel이 자동 감지 → **1~2분 후 배포 완료**

**내가 직접 할 명령어는 없습니다.**

## 5. 수정 요청 시 팁

### 좋은 요청

```
✅ "근로제등록.html의 표준근로제 고유 필드 섹션에서
    근무 시간 default를 09:00~18:00에서 08:30~17:30으로 변경해줘"

✅ "외근유형관리.html에 '해외 외근' 유형 설명을 추가해줘.
    기존 외근과 동일한 구조인데 출장 인정 단위가 '기간'으로 고정되는 점이 다르다고 써줘"

✅ "용어집에 새 용어 3개 추가해줘: 탄력근로제, 재량근로제, 사업장밖간주근로제"
```

### 주의사항

```
⚠️ 수정 전에 기존 매뉴얼 스타일을 유지하라고 말해주세요
   → "@CLAUDE.md 참고해서 기존 스타일 유지해줘"

⚠️ 새 페이지를 만들면 index.html에도 추가해야 합니다
   → "새 페이지 만들고 index.html에도 서브탭 추가해줘"

⚠️ 새 전문 용어가 나오면 용어집도 업데이트
   → "용어집에도 새 용어 추가해줘"
```

---

## 6. 다른 팀원과 동시 작업 시

동시에 같은 파일을 수정하면 충돌이 발생할 수 있습니다.

**예방법:**
- 작업 시작 전 Claude에게 `"git pull 먼저 해줘"` 라고 요청
- 가능하면 같은 파일을 동시에 수정하지 않기
- 충돌이 나면 Claude에게 `"충돌 해결해줘"` 라고 요청

## 8. 문제 해결

| 문제 | 해결 |
|------|------|
| GitHub 접근 안 됨 | Collaborator 초대를 수락했는지 확인 |
| push 후 배포 안 됨 | 1~2분 기다리기. 안 되면 관리자에게 문의 |
| Claude가 스타일 다르게 수정 | "@CLAUDE.md 참고해서 수정해줘" 추가 |
| 충돌 발생 | Claude에게 "충돌 해결해줘" 요청 |
| 어떤 파일을 수정해야 할지 모름 | Claude에게 "OO 내용이 어느 파일에 있어?" 먼저 물어보기 |
