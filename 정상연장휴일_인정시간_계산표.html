정상·연장·휴일 인정시간 상세 계산표

이 문서는 정상근무, 연장근무, 휴일근무의 인정시간을 계산할 때 실제로 어떤 설정값이 먼저 적용되고, 출퇴근 방식과 연장 인정 방식에 따라 어떤 차이가 생기는지를 표 중심으로 정리한 운영 매뉴얼입니다. 통합로직이 상위 흐름을 설명한다면, 이 문서는 그 흐름을 실제 계산 기준으로 내려서 보는 참조표입니다.

문서 개요

이 문서가 다루는 범위

정상근무, 연장근무, 휴일근무의 본시간 계산과 연장 인정 분기를 다룹니다. 출퇴근 방식, 근무패턴, 지각·조퇴 유예, 법정 최소 휴게, 결근 처리까지 포함해 일별 인정시간의 뼈대를 설명합니다.

이 문서를 보는 시점

근로제와 근무패턴, 근태신청, 출퇴근 저장이 모두 준비된 뒤에 어떤 값이 실제 인정시간으로 반영되는지 확인할 때 참고합니다. 화면별 안내보다 계산 기준 자체를 확인할 때 유용합니다.

핵심 요약

기준근무시간은 근로제와 근무패턴에서 먼저 잡히고, 그 위에 출퇴근 방식, 연장 인정 방식, 휴게와 유예시간 규칙이 겹쳐지면서 최종 인정시간이 정해집니다. 패턴이 등록된 날에는 패턴 자체보다 근태신청이 우선한다는 점도 함께 봐야 합니다.

기본 계산 전제

구분 기준 운영 해석
기준근무시간 근로제 시간, 근무패턴 시간 일별 인정시간 계산의 시작점입니다. 패턴이 없으면 근로제 시간, 패턴이 있으면 패턴 규칙이 기준시간을 가감하거나 근무·휴무 자체를 바꿀 수 있습니다.
실적 반영 출퇴근 저장, 외출/복귀, 전자결재 실제 저장된 출퇴근 시각은 기준시간 위에서 비교·보정됩니다. 다만 승인된 근태신청이 있으면 해당 시간대는 신청 기준이 먼저 반영됩니다.
휴게 실제 휴게 또는 법정 최소 휴게 인정근로시간은 휴게를 제외하고 계산합니다. 실제 휴게가 없더라도 법정 최소 휴게가 적용되는 구조인지 반드시 확인해야 합니다.
결근 근무의무일 + 타각 없음 + 인정 기안 없음 이 문서에서는 결근 시간을 별도 가산하지 않고 0시간 처리 기준으로 봅니다. 다만 결근 상태 확정 전에는 미확정·보정대기 상태를 둘 수 있습니다.

출퇴근 방식별 계산

정상·연장·휴일 계산에서 가장 먼저 갈리는 실무 분기 중 하나가 출퇴근 방식입니다. 같은 근로제라도 자동 등록인지, 실시간 등록인지에 따라 인정시간을 만드는 방식이 달라집니다.

출퇴근 방식 계산 출발점 인정시간 반영 방식 주의할 점
자동 등록 근로제 또는 패턴 기준시간 실제 타각보다 미리 정해진 근무 프레임을 먼저 기준으로 사용합니다. 타각은 이상 여부 확인이나 예외 검토 자료로 더 많이 쓰입니다. 운영은 편하지만 실제 체류시간과 차이가 날 수 있으므로, 지각·조퇴·결근 예외 처리 정책이 함께 있어야 합니다.
실시간 등록 실제 저장된 출퇴근 시각 실제 출퇴근 시각을 기준으로 근로제 시간대와 겹치는 범위를 본시간으로 잡고, 초과분은 연장 조건에 따라 나눕니다. 단순 체류시간을 전부 인정하는 것이 아니라, 근로제 시간대와의 교집합과 승인조건을 함께 봐야 합니다.

운영 포인트

자동 등록은 기준근무시간 중심, 실시간 등록은 실제 저장시간 중심으로 읽으면 이해가 쉽습니다. 다만 어느 방식이든 승인된 근태신청이 있으면 그 시간대는 신청 기준이 먼저 적용될 수 있습니다.

연장 인정 방식 비교

연장근무는 단순히 “소정시간을 넘었는가”만으로 끝나지 않습니다. 연장 인정 기능을 쓰지 않는지, 전자결재 승인 시간으로 볼지, 자동 인정으로 볼지에 따라 계산 버킷이 달라집니다.

연장 인정 방식 기준 본시간/연장시간 반영 운영 해석
미사용 연장 자동 인정 기능을 사용하지 않음 본시간은 근로제 또는 실시간 등록 기준으로 반영하고, 초과 체류시간은 별도 검토 대상으로 남깁니다. 무조건 0으로 버리기보다, 승인 누락인지 실제 초과근무인지 검토하는 절차가 필요합니다.
전자결재 승인 시간 인정 최종 승인된 신청시간과 실제 근무시간 비교 연장시간 버킷은 승인 기준으로 반영하되, 실제 근무시간이 더 짧거나 길면 세부 분기 규칙이 추가로 필요합니다. 이 방식은 승인시간을 존중하지만, 실제 근무와 불일치하면 보완안의 세부 분기 규칙까지 함께 봐야 합니다.
연장근무 자동 인정 근로제 기준시간을 넘긴 실제 저장시간 초과분을 시스템이 자동으로 연장시간으로 분리합니다. 실무상 빠르지만, 휴게·지각·조퇴·패턴 단축근무 같은 요소를 같이 고려하지 않으면 과대 산정될 수 있습니다.

전자결재 승인 시간 인정의 세부 읽는 법

세부 분기 설명 매뉴얼 해석
최종 승인 건 근로시간 적용 전자결재가 최종 승인된 시간 범위를 우선 사용 결재 완료된 시간대를 기준 버킷으로 잡되, 실제 근무 여부 확인은 따로 남길 수 있습니다.
실 근무시간 적용 승인시간보다 실제 근무시간을 더 강하게 반영 승인이 있어도 실제 근무가 짧으면 짧은 쪽을 반영하는 구조로 읽을 수 있습니다.
신청시간 이내 실적이 승인시간 범위 안에서 끝남 승인시간 전체를 채우지 못한 경우, 실제 인정시간을 어디까지 볼지 회사정책이 필요합니다.
신청시간 초과 실적이 승인시간을 초과함 초과분을 자동 인정할지, 추가 승인 대상으로 돌릴지 정책을 분리해야 합니다.

세부 정책 반영

정책 항목 반영 위치 계산에 미치는 영향
지각/조퇴 유예시간 정상근무 본시간 산정 전 몇 분 이내 지각·조퇴를 즉시 차감하지 않는 구조를 둘 수 있습니다. 다만 유예가 본시간 유지인지, 이상만 해제하는지 구분이 필요합니다.
법정 최소 휴게시간 일별 인정시간 계산 단계 실제 휴게가 별도 저장되지 않더라도 최소 휴게시간을 차감할 수 있습니다. 연장 자동 인정과 함께 쓰일 때 과대 산정을 막는 역할을 합니다.
결근 = 0시간 일별 결과 확정 단계 근무의무일인데 실적과 인정 기안이 모두 없으면 인정시간은 0시간으로 수렴합니다. 결근 여부 확정은 운영 검토 후 별도 상태로 관리할 수 있습니다.
휴일근무 인정 캘린더/휴일 판정 후 휴일 여부가 먼저 정해진 뒤, 실제 저장시간과 승인 정책에 따라 휴일근무 버킷이 만들어집니다.

근무패턴과 근태신청 우선

근무패턴이 등록되어 있어도 항상 패턴만으로 계산하지는 않습니다. 스프레드시트 기준으로는 패턴 등록 상태에서 근태신청이 존재하면 근태신청을 우선하는 구조를 두는 것이 맞습니다.

상황 우선 기준 해석
근무패턴 없음 근로제 기준 일반적인 기준근무시간 계산입니다.
근무패턴 있음, 근태신청 없음 패턴 기준 추가 근무, 단축 근무, 휴무 전환 등 패턴 규칙을 우선 반영합니다.
근무패턴 있음, 근태신청 있음 근태신청 우선 해당 시간대는 연차, 반차, 외근, 출장 같은 신청이 패턴보다 앞서 적용됩니다.

추가 근무 / 단축 근무 읽는 법

패턴은 단순히 예상 프레임을 그리는 수준이 아니라 기준근무시간 자체를 바꿀 수 있습니다. 따라서 추가 근무 = 근로제 시간 + 패턴 시간, 단축 근무 = 근로제 시간 - 패턴 시간이라는 분기를 통합로직과 함께 읽어야 합니다.

대표 예시

사례 설정 계산 해석
표준근무 + 자동 등록 근로제 8시간, 패턴 없음, 자동 등록 기준근무시간 8시간을 우선 사용합니다. 타각은 이상 여부나 예외 보정 자료로 확인합니다.
표준근무 + 실시간 등록 + 연장 자동 인정 실제 09:00~20:00, 휴게 차감, 자동 인정 사용 근로제 기준시간을 넘는 구간은 자동으로 연장시간 버킷으로 분리됩니다.
패턴 단축근무 + 근태신청 존재 패턴으로 2시간 단축, 오후 반차 승인 단축 패턴보다 근태신청이 우선입니다. 반차 시간대는 신청 기준으로 처리하고, 남은 시간에 대해서만 패턴 영향을 봅니다.
근무의무일 + 무타각 + 기안 없음 근무일, 근무패턴 없음, 실적 없음 결근 또는 미확정 근태 후보이며, 인정시간은 0시간 기준으로 봅니다.
