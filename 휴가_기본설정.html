휴가 기본설정

연차 관리의 핵심 정책을 설정하는 화면입니다. 마이너스 연차 허용, 휴가 자동부여, 생성기준(회계연도/입사일), 비례휴가 소수점 처리, 근속연수별 부여 설정, 자동이월 여부를 관리합니다.

경로: 근태관리 > 휴가 > 휴가 설정 > 기본설정 권한: 관리자

화면 개요

마이너스 연차

잔여 연차가 0인 상태에서도 휴가를 신청할 수 있도록 허용하는 설정입니다.

사용 (default)

잔여 연차가 0이어도 최대허용개수 범위 내에서 휴가 상신이 가능합니다. 익년도 연차를 미리 사용하는 개념입니다.

미사용

잔여 연차가 0이면 휴가 상신이 차단됩니다.

최대허용개수 마이너스 연차를 "사용"으로 설정했을 때 노출되는 숫자 입력 필드입니다. 기본값은 1이며, 최대 2자리 숫자까지 입력할 수 있습니다. 설정된 개수만큼 잔여 연차가 마이너스가 될 수 있습니다.
마이너스 연차로 사용한 일수는 다음 연도 연차 부여 시 자동으로 차감됩니다. 퇴직 시 미정산된 마이너스 연차는 급여에서 공제 처리됩니다.

휴가 자동부여

근속연수에 따라 연차를 자동으로 생성·부여하는 기능입니다.

미사용 (default)

담당자가 휴가부여·조정 메뉴에서 수동으로 연차를 부여합니다. 자동부여 기준 버튼이 비활성화됩니다.

사용

아래 기준에 따라 시스템이 자동으로 연차를 생성합니다.

자동부여 기준

1년 미만 근로자 입사일 기준 익월 1일 00시에 자동으로 1개씩 생성합니다. 매월 1개씩, 최대 11개까지 부여됩니다.
1년 이상 근로자 회계연도 기준 매년 1월 1일 00시에 근속연수에 따른 연차가 자동 생성됩니다. 기본 15일 + 가산휴가 + 이월분이 합산되어 부여됩니다.
대상 재직 중인 근로자만 자동부여 대상에 포함됩니다. 퇴직·휴직 상태인 근로자는 제외됩니다.

미사용에서 사용으로 변경 시 자동부여 로직

자동부여를 "미사용"에서 "사용"으로 변경하면, 변경 시점을 기준으로 아직 부여되지 않은 연차를 소급하여 자동 생성합니다.

입사일 기준 예시

2025년 7월 1일 입사자가 있고, 10월에 자동부여를 켰다면 → 8월분(1개), 9월분(1개), 10월분(1개)이 일괄 생성됩니다.

회계연도 기준 예시

2026년 1월 1일에 이미 자동부여 시점이 지난 1년 이상 근로자가 있다면 → 해당 연도의 기본연차 + 가산휴가가 일괄 생성됩니다.

자동부여 전환 시 이미 수동으로 부여된 연차가 있으면 중복 부여되지 않도록 기존 부여 이력을 확인한 뒤 처리됩니다.

휴가 생성기준

연차가 새로 생성되는 기준 시점을 선택합니다. 회사의 연차 관리 정책에 따라 두 가지 방식 중 하나를 선택합니다.

회계연도 (default)

매년 1월 1일 00시에 전 직원의 신규 연차를 일괄 생성합니다.

입사일자

근로자별 입사일자 당일 00시에 개인별로 신규 연차를 생성합니다.

회계연도 기준 상세

1년 이상 근로자 기본 15일 + 근속연수별 가산휴가 + 전년도 이월분을 합산하여 1월 1일에 부여합니다.
1년 미만 근로자 전년도 근속기간에 비례한 비례연차 + 전년도 이월분을 합산하여 1월 1일에 부여합니다.

입사일자 기준 상세

1년 이상 근로자 입사일자 도래 시 기본 15일 + 근속연수별 가산휴가 + 이월분을 합산하여 부여합니다.
1년 미만 근로자 매월 월차 1개씩 자동 생성됩니다. 입사 1년이 도래하면 기본 15일 + 이월분으로 전환됩니다. 1년 미만 기간에 사용한 월차는 15일에서 차감됩니다.
휴가 생성기준은 한 번 설정하면 운영 중 변경 시 기존 부여 이력과 충돌이 발생할 수 있으므로, 서비스 초기에 확정하는 것을 권장합니다.

비례휴가 생성 기준설정

1년 미만 근로자에 대해 회계연도 기준으로 연차를 생성할 때, 비례연차의 소수점 처리 방식을 설정합니다.

올림 (default)

소수점이 발생하면 무조건 올림 처리합니다.

반올림

소수점 이하를 반올림 처리합니다.

유지

소수점을 그대로 유지합니다.

비례연차 산출 공식

전년도 근속기간 비례분 = (입사년 재직일 / 365일) x 15일

계산 예시

2025년 7월 1일 입사자

2026년 1월 1일 비례연차 산출:
재직일수 = 2025.7.1 ~ 2025.12.31 = 184일
비례연차 = (184 / 365) x 15 = 7.56일

올림 적용 시 → 8일 부여
반올림 적용 시 → 8일 부여
유지 적용 시 → 7.56일 부여

비례휴가 설정은 "휴가 생성기준"이 회계연도인 경우에만 적용됩니다. 입사일자 기준에서는 월차 방식으로 운영되므로 이 설정이 영향을 주지 않습니다.

근속연수별 휴가부여 설정

근속연수에 따라 부여되는 연차 일수를 관리하는 테이블입니다. 기본 테이블은 근로기준법에 따라 자동 구성되며 삭제할 수 없고, 휴가일수와 비고만 수정할 수 있습니다.

가산휴가 공식

3년 차부터 가산: 15 + {(근속연수 - 1) / 2}일 (소수점 버림), 최대 25일
근속연수휴가일수산출근거
1년15일기본 연차
2년15일기본 연차
3년16일15 + (3-1)/2 = 15 + 1
4년16일15 + (4-1)/2 = 15 + 1 (소수점 버림)
5년17일15 + (5-1)/2 = 15 + 2
.........
21년 이상25일최대 한도
  • 수정 버튼을 클릭해야 휴가일수와 비고 컬럼이 활성화됩니다.
  • 회사 내규에 따라 법정 기준보다 더 많은 일수를 설정할 수 있습니다.
  • 기본 테이블의 행은 삭제할 수 없습니다.
근속연수별 휴가일수를 법정 기준보다 적게 설정하면 근로기준법 위반에 해당할 수 있습니다. 반드시 법정 기준 이상으로 설정하십시오.

휴가 자동이월 여부

전년도 미사용 연차를 다음 연도로 자동 이월할지 여부를 설정합니다.

미사용 (default)

자동이월이 되지 않습니다. 휴가이월·정산 메뉴에서 담당자가 수동으로 이월 처리해야 합니다.

사용

유급휴가만 자동이월됩니다. 무급휴가는 유효기간에 따라 소멸됩니다.

자동이월 적용 규칙

유급휴가 전년도 미사용분이 다음 연도 생성 시 자동으로 합산됩니다.
무급휴가 자동이월 대상이 아닙니다. 휴가유형에 설정된 유효기간에 따라 자동 소멸됩니다.

미사용에서 사용으로 변경 시 이월 로직

입사일 기준

변경 시점 이후 도래하는 입사일자에 맞춰, 전년도 미사용 유급휴가가 자동이월 처리됩니다. 이미 입사일이 지난 근로자는 다음 입사일 도래 시 적용됩니다.

회계연도 기준

변경 시점 이후 도래하는 1월 1일에 전년도 미사용 유급휴가가 자동이월 처리됩니다. 이미 1월 1일이 지난 상태라면 다음 연도 1월 1일에 적용됩니다.

자동이월 전환 시 기존에 이미 소멸 처리된 연차는 복구되지 않습니다. 전환 이후 새로 도래하는 시점부터 적용됩니다.

저장

설정 변경 후 저장 버튼을 클릭하면 확인 메시지가 표시됩니다.

  • 수정사항이 있는 경우: "설정한 값을 저장하시겠습니까? 저장 시 업무설정 적용을 위해 실행 중인 모든 메뉴가 종료됩니다."
  • 수정사항이 없는 경우: "설정한 값을 저장하시겠습니까?"
저장 시 모든 탭이 닫히고 메인 화면으로 이동합니다. 저장 전에 모든 설정값을 반드시 확인하십시오.
