인정근로시간을 한눈에 보는 통합 안내 페이지

이 페이지는 현재 근태관리 서비스에서 인정근로시간이 어떻게 만들어지고 월 근태 마감으로 넘어가는지를 한 장에서 보도록 정리한 문서입니다. 상단은 플로우맵 중심, 하단은 처리 구조와 산식, 우선순위, 추가 확인 항목 중심으로 구성했습니다.

기준: 현재 화면 구조와 계산 기준 정리 핵심 범위: 일별 판정 → 분류 데이터 → 월마감 → 급여 반영 자료 페이지 성격: 흐름 안내 + 계산 설명 + 추가 확인 항목
기준 정리 추가 확인 필요
인덱스로 돌아가기 → 기존 월마감 설명 보기 → 우선순위·충돌규칙 보완안 보기 → 정상·연장·휴일 계산표 보기 → 출장·외근 계산표 보기 → 통합 최적안 보기 →
목차
한눈에 보는 플로우맵 프로세스 구조 기준근무시간 산정 분기 출퇴근 저장 후 반영 분기 출장·외근 신청 반영 분기 출퇴근 방식과 상세 정책 반영 입력원천 우선순위 기준정보별 역할 핵심 산식 분류 결과 구조 유연근무와 정산 주기 운영자가 봐야 할 결과 테이블 구조 추가 확인 항목 결론

한눈에 보는 플로우맵

근로제 설정 → 근무패턴 여부 판단 → 기준근무시간 산정 → 예상 출퇴근 시간 생성 → 직원 출/퇴근 저장 → 일별 반영 분기 → 집계마감 기준데이터 생성 흐름에 월 집계와 급여 반영까지 연결한 통합 플로우입니다.

1. 기준 설정

근로제, 근무패턴, 캘린더, 휴가유형, 출장유형, 연장 인정 방식을 먼저 확정합니다.

  • 개인별 근로제
  • 근무요일/소정근로시간
  • 출퇴근 방식 설정
  • 휴일/대체휴일
2. 기준시간 산정

근무패턴 사용 여부를 판단해 해당 일자의 기준근무시간을 정합니다. 패턴에 따라 추가 근무와 단축 근무가 반영됩니다.

  • 패턴 미사용: 근로제 시간
  • 추가 근무: 근로제 시간 + 패턴 시간
  • 단축 근무: 근로제 시간 - 패턴 시간
3. 예상 프레임 생성

산정된 기준근무시간을 바탕으로 직원별 예상 출퇴근 시간과 예상 휴게구간을 만듭니다. 이 값이 정상/연장/이상 판정의 비교 프레임입니다.

  • 예상 출근 시각
  • 예상 퇴근 시각
  • 예상 휴게구간
4. 실적 저장

직원 출/퇴근, 외출, 휴게, 위치/인증방식, 근태 요청, 승인 결과를 저장하고 기본 실적을 만듭니다.

  • 출근/퇴근
  • 자동 등록 / 실시간 등록
  • 외출/복귀
  • 근태 요청/승인
5. 일별 반영 분기

휴가, 출장·외근, 출퇴근 인정 여부, 연장 자동 인정, 전자결재 인정 여부를 읽어 근무시간 또는 연장시간 반영 경로를 결정합니다.

  • 근무 시간 반영
  • 연장 근무 시간 반영
  • 패턴 등록 상태에서는 근태신청 우선
  • 근태 이상 분류
일별 판정
6. 월마감 집계

일별 결과를 월 단위로 합산하고 주 52시간, 선택근무 정산, 보정 여부를 반영합니다.

  • 월 집계
  • 유연근무 정산
  • 관리자 보정
월 집계 기준 유연근무 세부 기준 추가 확인
7. 확정 결과

월마감 결과를 급여 반영용 확정 자료로 만들고, 이후는 확정 결과를 기준으로 봅니다.

  • 월마감 결과
  • 급여 반영 자료
  • 재마감/소급 정책
급여 연계 소급 정책 추가 확인

패턴 기반 기준시간 분기

FigJam 기준으로 근무패턴은 단순 비교 프레임이 아니라 해당 일자의 기준근무시간을 가감하는 요소로도 작동합니다.

연장 인정 두 경로

직원 출/퇴근 저장 이후 연장근무는 자동 인정 경로와 전자결재 인정 경로로 나뉘며, 본시간 반영과 연장시간 반영을 분리해 봐야 합니다.

출장·외근 시간 기준 분기

출장·외근은 항상 같은 방식으로 반영되지 않고, 출퇴근 인정 여부와 유형 설정에 따라 근무시간기준 또는 신청시간기준으로 나뉩니다.

근태 이상 처리

근태 이상은 단순 오류가 아니라, 본시간 반영 여부와 관리자 검토 여부를 분리해서 봐야 하는 예외 상태입니다.

프로세스 구조

기준 데이터 레이어

근로제, 근무패턴, 개인 캘린더, 휴가유형, 출장유형, 출퇴근 방식처럼 계산의 기준이 되는 기준정보 단계입니다. 특히 FigJam 기준으로 근무패턴은 예상 프레임뿐 아니라 기준근무시간 산정에도 영향을 줍니다.

  • 근로제 유형: 일반 / 시차 / 자유 / 스케줄
  • 출퇴근 방식: 자동 등록 / 실시간 등록
  • 근무패턴: 기준근무시간 가감 + 예상 출퇴근 프레임 생성 기준
  • 휴가/출장 유형: 간주 및 차감 정책

일별 판정 레이어

실적, 승인 데이터, 유형 정책을 조합해서 하루의 인정근로시간과 이상 상태를 만드는 계산 레이어입니다. 출퇴근 저장 이후 정상 반영, 근태 이상, 연장 자동 인정, 전자결재 인정이 다시 갈라지고, 패턴 등록 상태에서는 근태신청 건을 우선 반영합니다.

  • 실제 타각
  • 승인된 근태 요청
  • 휴게/외출/간주구간 반영

월 확정 레이어

일별 결과를 월 단위로 집계하고 보정, 검토, 확정해서 급여 반영 자료를 생성하는 마감 단계입니다.

  • 월 집계
  • 관리자 검토 및 재마감
  • 급여 반영 자료

기준근무시간 산정 분기

FigJam 기준으로 기준근무시간은 모든 경우에 동일하게 생성되지 않습니다. 먼저 근무패턴 사용 여부를 판단하고, 패턴이 있는 경우에는 추가 근무인지 단축 근무인지에 따라 해당 일자의 기준근무시간이 다시 계산됩니다.

분기 기준시간 계산 업무 의미
패턴 미사용 근로제 시간 기준 기본 근로제에서 정한 소정근로시간을 그대로 사용합니다.
패턴 사용 + 추가 근무 근로제 시간 + 패턴 근무 시간 특정 일자에 추가 근무가 예정된 날로 보고 기준근무시간을 늘립니다.
패턴 사용 + 단축 근무 근로제 시간 - 패턴 근무 시간 특정 일자에 단축 근무가 예정된 날로 보고 기준근무시간을 줄입니다.
즉 근무패턴은 단순히 예외 일정을 표시하는 것이 아니라, 해당 일자의 기준근무시간과 예상 출퇴근 시간의 출발값 자체를 바꾸는 요소로 이해하는 것이 맞습니다.

출퇴근 저장 후 반영 분기

직원 출/퇴근이 저장된 뒤에는 단순히 실적만 쌓이는 것이 아니라, 어떤 경로로 시간을 반영할지 다시 분기됩니다. FigJam 기준으로 이 구간은 정상 저장, 근태 이상, 연장근무 자동 인정, 연장근무 전자결재 인정으로 나뉩니다.

상태 반영 경로 업무 의미
직원 출/퇴근 정상 저장 근무 시간 반영 정상 실적으로 보고 본시간 계산에 반영합니다.
근태 이상 검토 대상 이상 상태로 먼저 분류하고, 본시간 반영 여부는 관리자 검토 기준이 필요합니다.
연장근무 자동 인정 근무 시간 반영 유형 설정에 따라 자동 인정되는 초과시간을 본시간 반영 경로에서 함께 처리합니다.
연장근무 전자결재 인정 연장 근무 시간 반영 전자결재 승인 기준으로 연장시간 버킷에 따로 반영합니다.
여기서 중요한 점은 연장근무 자동 인정과 연장근무 전자결재 인정이 같은 방식으로 반영되지 않는다는 것입니다. 자동 인정은 본시간 반영 경로와 가깝고, 전자결재 인정은 연장시간 반영 경로와 가깝습니다.

출장·외근 신청 반영 분기

출장·외근 신청은 단순히 타각이 없는 시간을 채우는 정도가 아니라, 출퇴근을 인정할지 여부와 어떤 시간 기준으로 근무를 계산할지까지 함께 결정합니다.

분기 기준 반영 방식 업무 의미
출/퇴근 인정 안 함 근무시간기준 근로제 기준시간 또는 유형 설정 기준으로 근무시간을 반영합니다.
출/퇴근 인정 신청시간기준 출장·외근 신청서에 승인된 시간을 기준으로 반영합니다.
연장/야간 자동 인정 근무 시간 또는 연장시간 추가 반영 유형 설정에서 자동 인정이 켜져 있으면 해당 시간을 다시 분해해 반영합니다.
연장/야간 미인정 본시간만 반영 출장·외근 자체는 인정하되, 연장과 야간은 별도 인정하지 않습니다.

출퇴근 방식과 상세 정책 반영

스프레드시트 기준으로 인정근로시간은 상위 흐름만으로 끝나지 않고, 출퇴근 방식과 근로제 상세 정책에 따라 실제 계산식이 다시 갈라집니다. 이 값들은 별도 예외가 아니라 공식 계산 기준으로 보는 것이 맞습니다.

정책 항목 설정값 반영 기준
출퇴근 방식 설정 자동 등록 / 실시간 등록 자동 등록은 승인 또는 설정 시간 기준으로 읽고, 실시간 등록은 실제 타각과 근로제 구간의 교집합을 더 강하게 반영합니다.
패턴 등록 상태에서 근태신청 존재 근태신청 우선 근무패턴이 있어도 해당 일자의 승인된 근태신청이 존재하면 그 신청 건을 우선해 판정합니다.
지각/조퇴 처리 유예시간 미사용 / 출근 인정 시간 사용 / 퇴근 인정 시간 사용 유예시간이 설정되면 지정된 범위 안의 지각 또는 조퇴를 정상 근로로 인정할 수 있습니다.
법정 최소 휴게 시간 지정시간 / 4시간마다 실제 휴게가 없으면 설정된 방식으로 최소 휴게를 산정해 차감합니다.
결근 근무 실적 없음 결근은 0시간으로 처리하며, 예외 사유가 있으면 관리자 보정이나 승인기안으로 대체합니다.
즉 현재 기준에서 자동 규칙은 막연한 보조 기능이 아니라, 출퇴근 방식, 근태신청 우선순위, 지각·조퇴 유예, 법정 최소 휴게, 결근 0시간처럼 실제 계산값을 바꾸는 공식 정책으로 보아야 합니다.

기준정보별 역할

근로제와 근무패턴

근로제는 소정근로시간과 근무요일, 근무유형을 정하고, 근무패턴은 예상 출퇴근 프레임을 만듭니다. FigJam 기준으로 근무패턴은 필요 시 기준근무시간을 더하거나 빼는 방식으로도 작동합니다. 패턴이 있는 날이라도 승인된 근태신청이 존재하면 해당 신청이 우선 반영됩니다.

  • 일반/시차/자유/스케줄 분기
  • 소정근로시간 정의
  • 예상 출근/퇴근/휴게 프레임
  • 추가 근무 / 단축 근무 반영
  • 패턴 등록 상태에서 근태신청 우선

휴가유형

유/무급, 일수/시간/미차감, 유효기간, 소요시간을 정의합니다. 월마감 시 휴가 상태와 차감량을 별도로 관리해야 한다는 신호입니다.

  • 유급휴가시간 / 무급휴가시간 분리
  • 시간차 휴가 반영
  • 미차감 휴가 별도 처리

출장·외근 유형

출퇴근 인정 여부, 근무 인정 시간, 휴게 인정 시간, 휴일 포함 여부, 연장·야간 인정 여부를 통해 간주근무 정책을 만듭니다. FigJam 기준으로는 같은 출장·외근이라도 근무시간기준과 신청시간기준으로 다시 나누어 반영합니다.

  • 타각 대체 가능 여부
  • 간주근무 시간 생성
  • 근무시간기준 / 신청시간기준
  • 휴일/연장/야간 분해 허용

근태 요청 관리

근로제 수정, 출근시간 수정, 퇴근시간 수정, 근태 수정 요청의 승인·거절 결과가 최종 판정에 반영됩니다.

  • 사후 정정 절차
  • 관리자 사유 기록
  • 마감 전 예외 정리 창구

핵심 산식

일별 기본식

판정대상시간 = 실제 타각구간 + 승인된 간주근무구간

인정근로시간 = 판정대상시간 - 휴게시간 - 제외시간 ± 관리자 확정보정

연장인정시간 = 인정근로시간 중 소정근로시간 초과분

야간해당시간 = 인정근로시간 중 22:00~06:00 구간

휴일인정시간 = 휴일 판정일의 인정근로시간

월 집계식

월 인정근로시간 = 일별 인정근로시간 합계

월 연장근로시간 = 일별 연장인정시간 합계

월 야간해당시간 = 일별 야간해당시간 합계

월 휴일근로시간 = 일별 휴일인정시간 합계

급여반영값 = 월마감 확정 결과 기준

중요: 총 인정근로시간과 가산수당용 시간은 분리해서 관리합니다. 야간은 총 시간을 한 번 더 더하는 개념이 아니라, 이미 인정된 시간 중 야간 구간이라는 속성입니다.

기준근무시간 = 근로제 시간

패턴 적용 기준근무시간 = 근로제 시간 ± 패턴 근무 시간

출장·외근 반영시간 = 근무시간기준 또는 신청시간기준

연장 자동 인정 = 근무 시간 반영 경로

연장 전자결재 인정 = 연장 근무 시간 반영 경로

결근 = 0시간

분류 결과 구조

분류 설명 어디에 쓰는가 출처 상태
정상 출퇴근 예상 출퇴근 프레임과 실제/승인 결과가 정상 범위 안에 들어온 일별 데이터입니다. 일별 기본 실적, 월 집계의 정상 기준 확인 완료

휴일 근무 휴일 판정일에 발생한 근무 데이터입니다. 휴일근로 집계와 가산 계산 확인 완료

유연근무와 정산 주기

현재까지 명확한 부분

일반/시차/자유/스케줄 근로제와 선택근무에 대한 주/월 정산 관점이 반영되어 있습니다. 선택근무와 유연근무는 정산기간 범위 판정이 필요합니다.

추가 확인이 필요한 부분

탄력근무, 선택근무, 자유근무 각각의 초과근로 판정식은 아직 수식 수준으로 모두 확정되지 않았습니다. 정산기간 기준 상세식, 법 적용 범위, 월마감 반영 위치는 추가 정리가 필요합니다.

항목 현재 상태 왜 보완이 필요한가
선택근무 / 탄력근무 상세 산식 방향은 정의되어 있으나 개념 표현 수준 월마감에서 정산기간 기준 초과근로 판정식을 명문화해야 합니다.
소급 수정 정책 필요성은 언급되어 있으나 명확한 처리화면은 아직 부족 월마감 이후 정정 요청이 들어왔을 때 재마감 / 소급반영 / 차월반영 정책이 필요합니다.
근태 이상 세부 분류코드 이상건 KPI는 존재하나 상세 코드표는 미확정 운영자 검토와 리포트 품질을 위해 이상유형 코드 체계가 필요합니다.
휴게 자동 차감 예외 중요 포인트로 언급되어 있으나 규칙 화면이 뚜렷하지 않음 직군별/유형별 휴게 예외 정책을 세부화해야 분쟁 위험을 줄일 수 있습니다.
출장 기간형 일자 분해 출장 인정 단위는 확인 완료, 일자별 분해식은 미확정 기간형 출장의 일별 인정시간 계산식을 따로 정의해야 합니다.
근태 이상의 본시간 반영 기준 근태 이상 분기 존재는 확인 완료, 본시간 반영 조건은 미확정 이상 상태가 즉시 본시간 반영인지, 관리자 검토 후 반영인지 명문화가 필요합니다.
연장 자동 인정과 전자결재 인정의 정산 차이 두 경로의 분기 존재는 확인 완료, 급여 및 정산 반영 차이는 미확정 자동 인정이 본시간 가산인지, 연장시간 버킷 분리인지 구체 규칙이 필요합니다.
출장·외근의 근무시간기준 / 신청시간기준 상세식 두 기준의 분기 존재는 확인 완료, 세부 계산식은 미확정 유형별 설정값과 승인시간을 어떻게 연결할지 계산식을 명확히 해야 합니다.

결론

현재 구조에서 인정근로시간은 출퇴근 타각만으로 계산되는 값이 아니라, 근로제와 근무패턴으로 만든 예상 프레임, 출퇴근 방식 설정, 휴가/출장 유형 정책, 승인 완료된 근태 요청, 실제 출퇴근 실적을 조합해서 만드는 일별 판정 결과입니다. 그리고 월 근태 마감은 이 일별 결과를 정상 출퇴근 / 연장 인정 / 근태 이상 / 휴일 근무로 분류한 뒤 집계, 검토, 확정해서 급여 반영 자료로 넘기는 구조로 이해하는 것이 가장 정확합니다.

즉 핵심 문장은 기준 설정 → 예상 프레임 → 실적 수집 → 정책 반영 → 일별 판정 → 분류 데이터 → 월마감 확정이며, 여기서 정책 반영에는 패턴 등록 상태에서의 근태신청 우선, 지각·조퇴 유예, 법정 최소 휴게, 결근 0시간이 포함됩니다.
