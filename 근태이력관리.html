근태 이력 관리

근태 관리 서비스에서 발생한 모든 변경 이력을 기간·기능·메뉴·처리자·대상자 기준으로 조회하는 감사(Audit) 화면입니다. 근로제 배포, 수정 요청 처리, 출퇴근 변경, 휴가 부여·조정 등 모든 근태 관리 작업의 이력을 추적할 수 있습니다.

경로: 근태관리 > 근태 이력 관리 권한: 관리자
목차
화면 개요 검색 영역 계단식 필터 구조 조회 영역 이력 추적 활용 시나리오 연결 문서

화면 개요

근태 이력 관리는 관리자 전용 조회 화면으로, 근태 관리 서비스에서 발생한 모든 변경 이력을 한곳에서 확인할 수 있습니다.

  • 수정/삭제 기능 없이 순수 이력 조회만 제공합니다.
  • 감사(Audit) 대응 및 변경 추적 목적으로 사용합니다.
  • 근로제 배포, 수정 요청 처리, 출퇴근 변경, 휴가 부여·조정 등 모든 근태 관리 작업의 이력이 자동으로 기록됩니다.
이 화면은 읽기 전용입니다. 이력 데이터를 직접 수정하거나 삭제할 수 없으며, 모든 변경 내역은 시스템에 의해 자동으로 기록됩니다.

검색 영역

조회 조건을 조합하여 원하는 이력을 검색합니다.

기간 from(시스템일자 - 1개월) ~ to(시스템일자)가 기본값입니다. 달력 아이콘을 클릭하여 날짜를 선택할 수 있습니다.
기능 드롭다운 (전체 default). 근태 관리 기능별로 조회합니다. 선택한 기능의 하위 메뉴만 메뉴 드롭다운에 노출됩니다.
메뉴 드롭다운 (전체 default). 기능에서 선택한 항목의 하위 메뉴만 노출됩니다. 계단식(cascade) 필터로 동작합니다.
구분 드롭다운 (전체 default). 메뉴에서 선택한 항목의 하위 구분만 노출됩니다.
처리자 근태 처리를 수행한 관리자 이름을 검색합니다. 인풋 10자, 한글/영문 입력 가능합니다.
ID 검색 시 20자, 영문/숫자/특수문자 입력 가능합니다.
대상자 처리 대상이 된 직원 이름을 검색합니다. 인풋 10자, 한글/영문 입력 가능합니다.
ID 검색 시 20자, 영문/숫자/특수문자 입력 가능합니다.
조회 버튼 Enter 키 또는 클릭으로 검색을 실행합니다. 결과가 없으면 "조회된 데이터가 없습니다" 메시지가 표시됩니다.

계단식 필터 구조

기능, 메뉴, 구분은 상위 선택에 따라 하위 옵션이 자동으로 필터링되는 계단식(cascade) 구조입니다.

1단계: 기능

최상위 분류를 선택합니다. 근무설정, 근태요청, 휴가관리, 근태결과 등 근태 관리 주요 기능 단위입니다.

2단계: 메뉴

선택한 기능의 하위 메뉴만 표시됩니다. 예: 기능 "근무설정" 선택 시 근로제 관리, 근무지 관리, 배포 등이 노출됩니다.

3단계: 구분

선택한 메뉴의 하위 구분만 표시됩니다. 예: 메뉴 "근로제 관리" 선택 시 등록, 수정, 삭제, 배포 등이 노출됩니다.

상위 항목을 변경하면 하위 항목은 자동으로 "전체"로 초기화됩니다. 예시: 기능 "근무설정" → 메뉴 "근로제 관리 / 근무지 관리 / 배포" → 구분 "등록 / 수정 / 삭제 / 배포"

조회 영역

검색 조건에 맞는 이력이 테이블 형태로 표시됩니다.

항목 설명
일시 처리 일시를 YYYY-MM-DD HH:MM:SS 형식으로 표시합니다.
처리자 관리자 이름(사번) 형태로 표시합니다.
대상자 처리 대상 직원 이름(사번) 형태로 표시합니다. N개 항목을 일괄 수정한 경우 "N개의 항목이 수정되었습니다" 형태로 요약됩니다.
기능 계단식 분류의 최상위 기능값입니다.
메뉴 계단식 분류의 중간 메뉴값입니다.
구분 계단식 분류의 하위 구분값입니다.
처리 내용 실제 변경 내용을 요약한 텍스트입니다. 변경 전·후 값이 포함될 수 있습니다.

이력 추적 활용 시나리오

근태 이력 관리 화면은 다음과 같은 상황에서 활용할 수 있습니다.

감사 대응

"누가, 언제, 어떤 근태 데이터를 변경했는지" 추적합니다. 내부 감사 또는 외부 감사 시 변경 이력 증빙 자료로 활용할 수 있습니다.

오류 추적

근태 계산 오류가 발생했을 때 원인이 되는 변경 이력을 파악합니다. 어떤 시점에 어떤 변경이 있었는지 역추적할 수 있습니다.

배포 이력

특정 직원에 대한 근로제·근무지 배포 변경 이력을 확인합니다. 배포 시점과 적용 내역을 추적할 수 있습니다.

수정 요청 처리 이력

승인/거절 처리 내역과 사유를 확인합니다. 수정 요청에 대한 처리 과정을 시간순으로 추적할 수 있습니다.
