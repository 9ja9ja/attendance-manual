근태 요청 관리

직원이 요청한 근로제 수정, 출퇴근 시간 수정, 근태 수정 내역을 관리자가 조회하고, 승인 또는 거절하는 화면입니다. 개별 처리와 일괄 확정이 모두 가능하며, 처리 후에는 복구할 수 없습니다.

경로: 근태관리 > 근태 요청 관리 권한: 관리자
목차
화면 개요 검색 영역 조회 영역 일괄 확정 버튼 요청 일괄 확정 팝업 개별 승인 팝업 개별 거절 팝업 얼럿/컨펌 메시지 정리 승인 후 영향 마감 영향 연결 문서

화면 개요

근태 요청 관리는 관리자 전용 화면입니다. 직원이 등록한 근로제 수정, 출퇴근 시간 수정, 근태 수정 요청이 이 화면에 집계되며, 관리자가 건별 또는 일괄로 승인/거절 처리를 수행합니다.

  • 요청 목록은 신청일자 최신순(오름차순)으로 조회됩니다.
  • 동일 직원, 동일 신청일자의 요청이 여러 건 존재하면 마지막 요청 건만 노출됩니다.
  • 처리(승인/거절) 완료된 건은 상태가 변경되며, 처리 후에는 복구할 수 없습니다.
이 화면은 단순 조회가 아닌 확정 처리 화면입니다. 승인/거절 저장 후에는 되돌릴 수 없으므로, 처리 전 요청 내역과 적용 요청 일자를 반드시 확인하십시오.

검색 영역

검색 드롭다운 전체(default) / 이름 / 사원번호 / 부서 / 직급 중 선택합니다. 선택한 기준에 따라 검색어 범위가 결정됩니다.
검색어 입력 공란(default). 모든 텍스트 입력 가능하며, 최대 50자까지 입력할 수 있습니다. LIKE %검색어% 방식으로 부분 일치 검색합니다.
검색 실행 Enter 키 또는 조회 버튼 클릭으로 검색을 실행합니다.
결과 없음 조회 결과가 없으면 목록 영역에 "조회된 데이터가 없습니다" 문구가 노출됩니다.

조회 영역

검색 결과는 아래 컬럼으로 구성된 테이블 형태로 노출됩니다.

컬럼 설명
체크박스 개별 요청 건을 선택합니다. 1개 이상 선택 시 일괄 확정 버튼이 활성화됩니다.
부서 요청 직원의 부서/팀명을 표시합니다. 1단계 상위 조직까지 함께 노출됩니다.
이름(사번) 직원 이름과 사원번호를 함께 표시합니다.
요청 종류 4가지 유형으로 구분되며, 유형에 따라 상세 문구 형식이 다릅니다.
  • 근로제 수정: "['요청월' 근로제 수정 신청] 기존 '%근로제명%'(근로제종류)에서 '%근로제명%'(근로제종류)으로 변경 요청"
  • 출근 시간 수정: "[요청일자] 기존 출근 시간 %시간%에서 %시간%으로 변경 요청(±차이)"
  • 퇴근 시간 수정: "[요청일자] 기존 퇴근 시간 %시간%에서 %시간%으로 변경 요청(±차이)"
  • 근태 수정: 출퇴근 시간 모두에 대한 수정 요청
요청 사유 직원이 입력한 변경 사유입니다. 관리자가 승인/거절 판단 시 참고합니다.
신청 일자 요청이 등록된 일시입니다. 목록은 이 값 기준으로 최신순 정렬됩니다.
적용 요청 일자 직원이 변경 적용을 희망하는 일자입니다. 승인 시 이 일자부터 변경 사항이 반영됩니다.
상태 대기 / 승인 / 거절 중 하나로 표시됩니다.
동일 직원, 동일 신청일자의 요청이 여러 건 존재하면 마지막 요청 건만 노출됩니다. 이 화면은 "요청 이력 전체"가 아닌 "현재 유효한 요청" 중심으로 운영됩니다.

일괄 확정 버튼

조회 영역의 체크박스에서 1개 이상의 요청 건을 선택하면 일괄 확정 버튼이 활성화됩니다. 선택하지 않은 상태에서는 비활성(disabled) 상태입니다.

일괄 확정

버튼 클릭 시 요청 일괄 확정 팝업이 노출됩니다. 선택한 건을 한 번에 승인 또는 거절 처리할 수 있습니다.

요청 일괄 확정 팝업

일괄 확정 버튼 클릭 시 노출되는 팝업입니다. 선택한 요청 건을 모두 승인 또는 거절할 수 있으며, 관리자 사유를 입력하여 저장합니다.

상단 안내: "근태 요청 관리에서 선택한 요청 사항을 모두 승인/거절할 수 있습니다. 저장 후 처리한 내용에 대해서는 복구할 수 없습니다."
선택 요청 건수 "N건" 형태로 현재 선택된 요청 건수를 표시합니다.
확정 처리 구분 라디오 버튼으로 승인(default) 또는 거절을 선택합니다. 하나의 팝업에서 일괄 승인과 일괄 거절을 모두 처리하는 구조입니다.
사유 인풋 필드. 모든 텍스트 입력 가능하며, 최대 50자까지 입력할 수 있습니다. 입력한 사유는 직원 조회 화면에서 확인할 수 있습니다.
취소 버튼 별도 동작 없이 팝업이 닫힙니다.
저장 버튼 사유 미입력 시 얼럿(A005) 노출. 정상 입력 시 컨펌(C001 또는 C002) 후 저장을 실행합니다. 저장 완료 시 토스트(A004) 노출.

저장 시 얼럿 분기

  • 사유 미입력: A005 얼럿 노출 후 저장 중단
  • 선택 건 전체가 적용 요청 일자 이내: C001 컨펌 노출 후 저장
  • 선택 건 중 적용 요청 일자 초과 건 포함 (승인 시): C002 컨펌 노출 -- 적용 요청일이 지나지 않은 건만 승인 가능함을 안내
  • 저장 완료: A004 토스트 노출

개별 승인 팝업

조회 목록에서 개별 요청 건의 승인 버튼을 클릭하면 노출되는 팝업입니다.

"근태 요청 관리에서 선택한 요청 사항을 승인합니다. 저장 후 처리한 내용에 대해서는 복구할 수 없습니다."
요청 직원 정보 해당 직원의 부서, 이름(사번), 요청 종류, 요청 내역을 표시합니다.
사유 인풋 필드. 최대 50자. 관리자가 승인 사유를 입력합니다. 직원 조회 화면에서 확인 가능합니다.
저장 사유 미입력 시 A005 얼럿. 정상 입력 시 승인 컨펌 후 저장, 완료 시 A004 토스트 노출.

개별 거절 팝업

조회 목록에서 개별 요청 건의 거절 버튼을 클릭하면 노출되는 팝업입니다.

거절 팝업 안내

"근태 요청 관리에서 선택한 요청 사항을 거절합니다." 적용 요청 일자가 이미 초과된 건의 경우 자동으로 관련 안내가 표시됩니다.

요청 직원 정보 해당 직원의 부서, 이름(사번), 요청 종류, 요청 내역을 표시합니다.
사유 인풋 필드. 최대 50자. 관리자가 거절 사유를 입력합니다. 직원 조회 화면에서 확인 가능합니다.
저장 사유 미입력 시 A005 얼럿. 정상 입력 시 거절 컨펌 후 저장, 완료 시 A004 토스트 노출.

얼럿/컨펌 메시지 정리

C001 "선택한 'N'건에 대하여 '구분값' 처리를 하시겠습니까? 저장 후에는 다시 복구할 수 없습니다."
C002 "적용 요청 일이 지나지 않은 'N'건에 대하여 '승인' 처리가 가능합니다. 저장 후에는 다시 복구할 수 없습니다."
A004 "저장되었습니다." (토스트)
A005 "처리 사유를 입력해주세요."
승인 컨펌 "해당 요청을 승인 처리를 하시겠습니까? 저장 후 다시 복구할 수 없습니다."
거절 컨펌 "해당 요청을 거절 처리를 하시겠습니까? 저장 후 다시 복구할 수 없습니다."

승인 후 영향

승인이 완료되면 요청 종류에 따라 실제 근태 데이터가 변경됩니다.

근로제 수정 승인

적용 요청 일자부터 해당 직원의 근로제가 변경됩니다. 변경된 근로제에 따라 소정근로시간, 예상 출퇴근 시간 등 판정 기준이 갱신됩니다.

출퇴근 시간 수정 승인

해당 일자의 출퇴근 데이터가 요청 값으로 업데이트됩니다. 지각/정상 판정, 인정근로시간이 재계산됩니다.

승인 결과는 직원의 나의 근태 현황 화면에서 확인할 수 있습니다. 관리자가 입력한 처리 사유도 직원에게 노출됩니다.

마감 영향

승인된 요청은 인정근로시간 계산과 월마감 결과에 직접 영향을 미칩니다.

  • 인정근로시간: 출퇴근 시간 수정 승인 건은 해당 일자의 인정근로시간에 반영되며, 연장/야간 구간 판정에도 영향을 줍니다.
  • 근로제 변경: 근로제 수정 승인 건은 적용 일자 이후의 소정근로시간 프레임 자체를 변경하므로, 일별 판정 기준이 달라집니다.
  • 월마감 전 확인: 미승인(대기) 상태의 요청이 남아 있으면 마감 결과가 달라질 수 있습니다. 마감 전 대기 건을 모두 처리하는 것을 권장합니다.
월마감 시점에는 승인 완료 데이터만 반영됩니다. 대기 상태 요청은 마감 결과에 포함되지 않으므로, 마감 전 일괄 확정 기능을 활용하여 미처리 건을 정리하십시오.
