# 근태 인덱스 메뉴 재편 검토 계획

## 1. 문서 목적

이 문서는 `근태관리/근태매뉴얼/index.html`의 현재 메뉴 구성을 다시 점검하고,  
좌측 메뉴 과밀 문제를 줄이기 위한 최종 재편 방향을 정리한 계획서다.

이번 검토의 목적은 다음과 같다.

- 현재 인덱스에 추가된 최종 메뉴 구성을 다시 확인한다.
- 좌측 메뉴가 너무 세분화된 부분과, 탭으로 묶는 것이 더 적합한 부분을 구분한다.
- 이미 작성된 상세 HTML은 유지하면서, 탐색 구조만 더 간결하게 재정리한다.
- 추후 `index.html` 개편 시 바로 적용할 수 있는 실행 순서를 만든다.

## 2. 현재 상태 점검 결과

### 2.1 현재 좌측 메뉴 그룹

현재 인덱스에는 아래 8개 그룹이 있다.

1. `Overview`
2. `Setup — 근로제`
3. `Setup — 근무지`
4. `Setup — 배포`
5. `Setup — 근무유형`
6. `Setup — 휴가`
7. `Operation`
8. `Closing`
9. `Appendix`

실제 버튼 수는 총 31개다.

### 2.2 그룹별 버튼 수

| 그룹 | 버튼 수 | 비고 |
|------|--------:|------|
| Overview | 1 | 적정 |
| Setup — 근로제 | 3 | 적정 |
| Setup — 근무지 | 2 | 적정 |
| Setup — 배포 | 1 | 단독 그룹으로는 약함 |
| Setup — 근무유형 | 5 | 다소 많음 |
| Setup — 휴가 | 2 | 적정 |
| Operation | 10 | 과다 |
| Closing | 4 | 적정 |
| Appendix | 3 | 적정 |

### 2.3 인덱스에 연결되지 않은 문서

현재 `근태매뉴얼` 폴더의 HTML 중 인덱스에 연결되지 않은 문서는 아래 3개다.

- `직원근태.html`
- `휴가관리.html`
- `출장외근유형관리.html`

즉, 현재 인덱스는 많은 문서를 보여주고 있지만 동시에 일부 핵심 요약 문서를 놓치고 있다.

## 3. 현재 구조의 장점

현재 메뉴 구조에도 좋은 점은 분명하다.

### 3.1 화면이 빠짐없이 드러난다

근로제, 근무지, 배포, 근무유형, 휴가, 요청, 마감 등 실제 업무 화면이 많이 드러나 있어서  
문서가 이미 충분히 쌓였다는 느낌을 준다.

### 3.2 상세 문서가 잘 분리되어 있다

근로제 목록과 등록, 근무지 목록과 설정, 인정근로시간과 보완안처럼  
문서 성격이 분리되어 있어 상세 설명을 쓰기 좋은 기반은 이미 갖추고 있다.

### 3.3 마감 영역과 참고 영역이 어느 정도 분리되어 있다

`Closing`과 `Appendix`가 따로 있어 마감과 참고자료를 구분하려는 방향 자체는 좋다.

---

## 4. 현재 구조의 문제점

### 4.1 좌측 메뉴가 업무군보다 상세 문서 위주다

현재는 사용자가 먼저 `어떤 업무를 보려는지`보다  
`어떤 개별 화면 문서를 볼지`를 바로 선택해야 한다.

즉, 인덱스가 `업무 허브`보다 `문서 목록`처럼 느껴진다.

### 4.2 Setup이 너무 잘게 쪼개져 있다

`근로제`, `근무지`, `배포`, `근무유형`, `휴가`가 모두 설정 성격인데  
좌측에서 따로따로 노출되다 보니 시각적으로 메뉴가 길어진다.

특히 `배포`는 단독 1개 문서라서, 별도 그룹으로 남기기보다 상위 설정군에 묶는 편이 자연스럽다.

### 4.3 Operation 그룹이 과밀하다

현재 `Operation`은 10개 버튼으로 가장 무겁다.

이 안에는 다음 성격이 혼재한다.

- 근태 요청/승인
- 근태 이력 조회
- 직원 개인 화면
- 휴가 부여/조정
- 연차 현황/촉진

즉, 운영처리와 휴가운영이 한 그룹에 같이 들어 있어 그룹 정체성이 약해진다.

### 4.4 일부 핵심 요약 문서가 인덱스에서 빠져 있다

현재 아래 문서는 폴더에 있지만 인덱스에 직접 연결되어 있지 않다.

- `직원근태.html`
- `휴가관리.html`
- `출장외근유형관리.html`

이 3개는 각각

- 직원 화면 의미 정리
- 휴가 정책 요약
- 출장/외근 정책 통합 요약

성격의 문서라서, 세부 화면 문서보다 상위 탭 문서로 활용할 가치가 높다.

---

## 5. 검토 결론

현재 구조는 `문서가 너무 많아서` 문제라기보다  
`문서 노출 계층이 너무 얕아서` 문제가 생긴 상태다.

즉:

- 상세 HTML은 유지해도 된다.
- 문제는 인덱스에서 그것들을 모두 좌측 메뉴로 직접 노출한 방식이다.

따라서 다음 방향이 적합하다.

1. 좌측 메뉴는 상위 업무군만 남긴다.
2. 상세 문서는 메인 영역 상단 탭으로 이동한다.
3. 이미 만든 HTML 파일은 삭제하지 않는다.
4. 누락된 요약 문서는 오히려 상위 탭 문서로 승격해 활용한다.

---

## 6. 최종 권장 IA

좌측 메뉴는 아래 6개로 재편하는 것을 권장한다.

1. `개요`
2. `기준설정`
3. `운영처리`
4. `휴가운영`
5. `마감정산`
6. `참고자료`

이 구조면 생산원가 매뉴얼처럼 상위 구조는 단순해지고,  
근태 특유의 세부 문서도 유지할 수 있다.

## 8. 메뉴 재편 우선순위

### 8.1 1차

좌측 메뉴에서 가장 먼저 줄여야 하는 부분은 설정군이다.

합치기 대상:

- `Setup — 근로제`
- `Setup — 근무지`
- `Setup — 배포`
- `Setup — 근무유형`
- `Setup — 휴가`

재편 결과:

- `기준설정`

### 8.2 2차

그 다음은 운영과 휴가운영을 분리한다.

현재 `Operation`에서 분리할 대상:

- `휴가부여조정.html`
- `휴가배포이력.html`
- `월별연차현황.html`
- `연차촉진관리.html`
- `연차촉진현황.html`

재편 결과:

- `운영처리`
- `휴가운영`

### 8.3 3차

마지막으로 마감과 별첨을 재정리한다.

합치기 대상:

- `Closing`
- `Appendix` 중 인정근로시간 관련 문서

재편 결과:

- `마감정산`
- `참고자료`

## 9. 실행 계획

### STEP 1. 인덱스 UI 구조 변경

- 좌측 메뉴를 6개 상위 그룹으로 재구성한다.
- 기존 세부 버튼은 좌측에서 제거한다.
- 메인 영역 상단에 그룹별 탭 바를 추가한다.

### STEP 2. 탭-iframe 연결

- 그룹 선택 시 해당 그룹의 탭 목록을 표시한다.
- 탭 클릭 시 기존 HTML 파일을 iframe에 로드한다.
- 현재의 `viewerTitle`, `viewerDescription`, `openPage` 로직은 유지하되 탭 기준으로 갱신한다.

### STEP 3. 누락 문서 편입

아래 3개 문서를 반드시 인덱스에 편입한다.

- `직원근태.html`
- `휴가관리.html`
- `출장외근유형관리.html`

### STEP 4. 설명 문구 정리

- descriptions 객체를 새 IA에 맞게 다시 정리한다.
- 그룹 설명과 탭 설명을 분리해서 관리할 수 있게 구조를 바꾸는 것이 좋다.

### STEP 5. 관련 문서 링크 점검

- 각 상세 HTML 하단의 관련 링크도 새 그룹 구조에 맞게 다시 점검한다.
- 특히 `직원근태`, `휴가관리`, `출장외근유형관리`는 상위 허브 문서 역할을 하도록 링크를 강화한다.

## 10. 완료 기준

다음 조건을 만족하면 메뉴 재편이 완료된 것으로 본다.

1. 좌측 메뉴가 6개 수준의 상위 업무군으로 줄어든다.
2. 세부 문서는 상단 탭으로 이동한다.
3. 현재 존재하는 HTML 문서가 인덱스에서 빠짐없이 접근 가능하다.
4. `Operation` 과밀 문제가 해소된다.
5. `휴가운영`과 `마감정산`의 업무 정체성이 더 분명해진다.
6. 신규 사용자가 처음 봐도 구조를 빠르게 이해할 수 있다.

---

## 11. 최종 의견

현재 메뉴는 문서가 잘 쌓인 상태라는 점에서는 좋다.  
다만 이제는 `문서를 더 추가하는 단계`보다 `탐색 구조를 정리하는 단계`로 넘어온 상태다.

따라서 다음 작업은 새 문서를 더 만드는 것보다,  
이미 만든 문서를 `상위 메뉴 + 탭 구조`로 재편해 근태매뉴얼 전체를 더 읽기 쉽게 만드는 데 초점을 맞추는 것이 맞다.
