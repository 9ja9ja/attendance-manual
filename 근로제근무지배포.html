근로제·근무지 배포

등록된 근로제와 근무지를 직원에게 배포하는 화면입니다. 조직도에서 직원을 선택하고, 개별 또는 일괄로 근로제와 근무지를 지정할 수 있습니다. 배포된 근로제·근무지는 직원의 예상 출퇴근 프레임 생성과 월마감 계산의 기준이 됩니다.

경로: 근태관리 > 설정 > 근로제·근무지 배포 권한: 최고 관리자 / 총괄 관리자
목차
1. 화면 개요 2. 조직도 영역 3. 검색 영역 4. 직원 조회 영역 5. 주요 버튼 6. 일괄 배포 팝업 7. 수정기간 설정 팝업 8. 배포 예정 리스트 9. 저장 동작 규칙 10. 마감 영향 11. 연결 문서

1. 화면 개요

근로제·근무지 배포 화면은 좌측 조직도 영역과 우측 직원 리스트 영역으로 구성됩니다.

  • 좌측: 계정이 속한 조직(부서) 트리를 계층 구조로 표시합니다.
  • 우측: 선택한 조직에 속한 직원 목록을 조회하고, 근로제와 근무지를 개별 또는 일괄로 배포합니다.
최초 진입 시 모든 조직이 펼쳐진 상태로 노출되며, 전체 직원이 default로 자동 조회됩니다.
이 화면은 최고 관리자 또는 총괄 관리자 권한이 있어야 접근할 수 있습니다.

2. 조직도 영역

화면 좌측에 위치하며, 계정이 속한 조직(부서)을 트리 구조로 표시합니다.

  • 조직 트리는 최상위 조직부터 하위 팀까지 계층적으로 펼쳐집니다.
  • 특정 부서 또는 팀을 선택하면, 해당 조직에 속한 직원이 우측 조회 영역에 노출됩니다.
  • 상위 조직을 선택하면 하위 조직의 직원까지 포함하여 조회됩니다.

3. 검색 영역

직원 리스트 상단에 위치한 검색 기능으로, 원하는 직원을 빠르게 찾을 수 있습니다.

검색 드롭다운 검색 범위를 선택합니다. 전체(default) / 이름 / 사원번호 / 부서 중 선택할 수 있습니다.
검색어 입력 검색할 키워드를 입력합니다. 기본값은 공란이며, LIKE %검색어% 방식으로 부분 일치 검색됩니다.
조회 실행 Enter 키 또는 조회 버튼을 클릭하여 검색을 실행합니다.
결과 없음 검색 결과가 없을 경우 "검색 결과가 없습니다." 안내 메시지가 노출됩니다.

4. 직원 조회 영역

조직도에서 선택한 부서/팀에 속한 직원 목록이 표시되는 핵심 영역입니다. 각 직원별로 근로제와 근무지를 직접 선택하여 개별 배포할 수 있습니다.

체크박스 직원을 선택합니다. 하나 이상 선택하면 상단의 일괄 배포 버튼이 활성화됩니다. 전체 선택 체크박스로 현재 조회된 전 직원을 한 번에 선택할 수 있습니다.
부서 직원이 소속된 부서를 표시합니다. 선택한 depth보다 1단계 상위 조직까지 함께 조회됩니다.
예: "2025년 Service 본부_Business팀"
직급 직원의 직급 정보를 표시합니다.
직책 직원의 직책 정보를 표시합니다.
이름(사번) 직원의 이름과 사원번호를 함께 표시합니다.
지정 일자 근로제 또는 근무지가 적용되는 일자(업데이트 일자)입니다. 근로제 지정 일자와 근무지 지정 일자 중 마지막(최신) 일자로 조회됩니다.
근로제명 드롭다운 형식으로 근로제를 선택합니다. 미사용 상태의 근로제는 드롭다운에 노출되지 않습니다.
· 신규 직원: 대표 근로제가 default로 선택됩니다.
· 기존 직원: 현재 배포된 근로제가 default로 선택됩니다.
근무지명 다중 선택 드롭다운 형식으로 근무지를 선택합니다. 미사용 상태의 근무지는 드롭다운에 노출되지 않습니다.
· 대표 근무지는 반드시 선택되어야 합니다.
· 근무지를 추가하거나 삭제할 수 있으며, 복수의 근무지를 지정할 수 있습니다.
변경 사유 근로제 또는 근무지를 변경한 사유를 입력합니다. 텍스트 입력 형식이며 최대 50자까지 입력할 수 있습니다.

5. 주요 버튼

수정기간 설정

직원이 근로제 수정을 요청할 수 있는 기간을 관리자가 설정합니다. 클릭 시 수정기간 설정 팝업이 열립니다.

일괄 배포

체크박스로 직원을 선택한 후 활성화됩니다. 다수의 직원에게 동일한 근로제와 근무지를 한 번에 배포합니다. 클릭 시 일괄 배포 팝업이 열립니다.

저장

개별 직원의 근로제/근무지 변경사항이 있을 때만 활성화됩니다. 저장 시 확인 팝업이 노출되며, 근무지가 선택되지 않은 직원이 있으면 얼럿이 표시됩니다.

엑셀 다운로드

현재 조회된 직원 리스트 정보를 엑셀 파일로 다운로드합니다.

엑셀 업로드

엑셀 양식을 이용하여 다수 직원의 근로제/근무지를 일괄 등록합니다.
· 필수 항목: 이름, 사원번호, 적용시점
· 선택 항목: 근로제명, 근무지명

6. 일괄 배포 팝업

체크박스로 직원을 선택한 후 일괄 배포 버튼을 클릭하면 열리는 팝업입니다. 상단의 배포 정보 입력 영역과 하단의 근로제/근무지 정보 탭으로 구성됩니다.

6-1. 일괄 배포 정보 영역 (상단)

선택 직원 선택한 직원 목록을 표시합니다. 1명이면 해당 직원의 이름이, 다수이면 "김대리 외 N명" 형식으로 표시됩니다.
적용 일자 근로제/근무지가 적용되는 날짜입니다. 기본값은 시스템 일자 +1일이며, 달력 아이콘으로 날짜를 선택합니다. 시스템 일자 이전의 날짜는 선택할 수 없습니다.
근로제 드롭다운 형식으로 근로제를 선택합니다. 기본값은 대표 근로제입니다. 하단 근로제 정보 탭에서 라디오 버튼으로 선택하면 자동으로 변경됩니다.
근무지 다중 선택 드롭다운 형식입니다. 하단 근무지 정보 탭에서 체크박스로 선택하면 자동으로 연동됩니다. 대표 근무지는 반드시 선택되어야 합니다.
변경 사유 변경 사유를 입력합니다. 최대 50자이며, 모든 텍스트를 입력할 수 있습니다.
즉시 체크박스 선택하면 적용 일자를 시스템 일자(오늘)로 설정하여 즉시 배포합니다.

6-2. 근로제 정보 탭 (하단 좌측)

등록된 근로제 리스트를 조회하는 영역입니다.

  • 조회 컬럼: 근로제명 / 사용근로제 / 근무요일 / 소정근로시간
  • 대표 근로제가 최상단에 위치하며, 이후 등록 순서대로 정렬됩니다.
  • 라디오 버튼으로 근로제를 선택하면 상단의 근로제 드롭다운에 자동 반영됩니다.

6-3. 근무지 정보 탭 (하단 우측)

등록된 근무지 리스트를 조회하는 영역입니다.

  • 조회 컬럼: 근무지명 / 근무지주소 / IP주소대역 / WIFI주소
  • 대표 근무지가 최상단에 위치하며, 이후 등록 순서대로 정렬됩니다.
  • 체크박스로 다중 선택하면 상단의 근무지 드롭다운에 자동 반영됩니다.
  • 대표 아이콘: 체크 선택된 근무지에 한해 대표 근무지를 지정할 수 있습니다. 대표로 지정된 근무지는 체크박스 해제가 불가능합니다.

6-4. 저장/취소 동작

저장 버튼 클릭 시 아래 얼럿이 조건에 따라 노출됩니다.

A001 저장되었습니다.
A002 배포하고자 하는 근로제/근무지 정보가 없습니다. 확인 후 배포해 주세요.
A003 %사원명%이 %적용일자%에 배포 예정인 대상에 포함되어 있습니다.
A004 근무지가 선택되지 않았습니다. 한개 이상의 근무지를 선택해주세요.

취소 버튼 클릭 시:

  • 변경사항이 있으면 "변경사항이 저장되지 않습니다. 팝업을 닫으시겠습니까?" 컨펌 팝업이 노출됩니다.
  • 변경사항이 없으면 팝업이 바로 닫힙니다.

7. 수정기간 설정 팝업

직원이 자신의 근로제 변경을 요청(수정)할 수 있는 기간을 관리자가 설정하는 팝업입니다. 4가지 수정기간 단위 중 하나를 선택합니다.

신청 기간 지정 일회성으로 수정 가능 기간을 설정합니다. 시작일은 시스템 일자, 종료일은 시스템 일자 +7일이 기본값입니다. 설정된 기간이 종료되면 직원 화면에서 수정 아이콘이 노출되지 않습니다.
매월 정기 신청 매월 반복되는 수정 가능 기간을 설정합니다. 매월 N일~N일 형식으로 입력하며, 1~31까지 지정할 수 있습니다. 31을 입력하면 해당 월의 말일로 처리됩니다.
상시 수정 직원이 항상 근로제 수정을 요청할 수 있습니다. 직원 화면에서 수정 아이콘이 항상 노출됩니다.
수정 불가 (default) 직원의 근로제 수정 요청을 차단합니다. 직원 화면에서 수정 아이콘이 노출되지 않습니다.

저장 시 아래 얼럿이 노출됩니다.

A004 저장되었습니다.

8. 배포 예정 리스트

배포 예정 정보가 있는 직원을 선택하면 화면 하단에 해당 직원의 배포 예정 리스트가 노출됩니다.

8-1. 조회 컬럼

체크박스 배포 예정 항목을 선택합니다. 하나 이상 선택하면 삭제 버튼이 활성화됩니다.
예정일자 근로제/근무지가 적용될 예정 날짜입니다.
대상직원 배포 대상 직원의 이름을 표시합니다.
근로제명 배포 예정인 근로제 이름입니다.
근무지명 배포 예정인 근무지 이름입니다.
변경사유 배포 시 입력한 변경 사유입니다.
상세 버튼 클릭 시 배포 예정 상세 조회 팝업이 열립니다. 직원별 배포 정보를 확인하고 개별 삭제가 가능합니다.

8-2. 검색 및 삭제

  • 검색: 대상직원 이름으로 검색합니다. 최대 10자이며 한글/영문을 입력할 수 있습니다.
  • 삭제: 체크박스로 항목을 선택한 후 삭제 버튼을 클릭합니다. "선택한 배포 예정 항목을 삭제하시겠습니까?" 컨펌 팝업이 노출됩니다.

8-3. 배포 예정 상세 조회 팝업

상세 버튼 클릭 시 열리는 팝업으로, 해당 직원의 배포 예정 정보를 상세하게 조회할 수 있습니다. 개별 배포 예정 항목을 삭제할 수 있습니다.

9. 저장 동작 규칙

직원 조회 영역에서 개별적으로 근로제/근무지를 변경한 후 저장하는 경우의 동작 규칙입니다.

변경 후 저장

변경사항이 있는 상태에서 저장 버튼 클릭 시 "저장하시겠습니까?" 확인 팝업이 노출됩니다.

부서 이동 시

변경사항이 있는 상태에서 다른 부서를 선택하면 "변경사항이 저장되지 않습니다. 이동하시겠습니까?" 컨펌 팝업이 노출됩니다.

근무지 미선택

근무지가 선택되지 않은 직원이 있는 상태에서 저장하면 "근무지가 선택되지 않았습니다." 얼럿이 표시됩니다.

변경사항이 없는 상태에서는 저장 버튼이 비활성화됩니다. 반드시 근로제 또는 근무지를 변경한 후 저장해야 합니다.

10. 마감 영향

근로제·근무지 배포는 직원의 근태 데이터 전반에 직접적인 영향을 미칩니다.

  • 예상 출퇴근 프레임 생성: 배포된 근로제의 근무요일과 소정근로시간을 기준으로 직원별 예상 출퇴근 프레임이 자동 생성됩니다. 근로제가 변경되면 적용 일자 이후의 프레임이 재생성됩니다.
  • 월마감 계산 기준: 인정근로시간 월마감 시 배포된 근로제의 소정근로시간과 근무지 정보를 기준으로 정상 출근/지각/조퇴/결근 등을 판정합니다.
  • 근무지 기반 출퇴근 인증: 배포된 근무지의 IP 주소 대역과 WIFI 주소를 기준으로 출퇴근 위치 인증이 수행됩니다.
근로제와 근무지가 정확하게 배포되지 않으면 예상 출퇴근 프레임이 생성되지 않아 월마감 계산에 오류가 발생할 수 있습니다. 신규 직원 입사 시 반드시 배포를 완료하십시오.
배포 예정 일자가 도래하면 시스템이 자동으로 해당 근로제·근무지를 적용합니다. 적용 전에 배포 예정 리스트에서 확인하고 필요 시 수정 또는 삭제할 수 있습니다.
