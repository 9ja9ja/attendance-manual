출장·외근 인정시간 상세 계산표

이 문서는 출장과 외근 유형 설정에 따라 인정시간이 어떻게 달라지는지를 정리한 계산표입니다. 특히 근무시간기준 / 신청시간기준, 근무시간 자동 인정, 연장·야간·휴일 인정 사용 여부를 실제 운영 언어로 풀어서 정리했습니다.

문서 개요

이 문서가 필요한 이유

출장과 외근은 둘 다 현장 밖 근무처럼 보이지만, 인정시간을 만드는 방식은 같지 않습니다. 어떤 유형은 근무시간을 자동 인정하고, 어떤 유형은 신청시간만큼만 반영하며, 또 어떤 유형은 연장·야간·휴일을 따로 인정합니다.

이 문서의 기준

유형 설정표와 기존 인정근로시간 문서에서 공통으로 읽히는 로직을 운영 기준으로 정리했습니다. 기획 메모에 남아 있는 수정 필요 항목은 따로 표시해 과도한 확정 해석을 피했습니다.

핵심 요약

출장·외근은 단순히 “타각이 없으니 근무 인정”으로 읽으면 안 됩니다. 유형별로 근무시간기준을 쓸지, 신청시간기준을 쓸지, 그리고 연장·야간·휴일을 따로 인정할지 여부가 갈립니다.

용어 통일

기존 시트와 화면 설명에는 출/퇴근 인정처럼 다소 넓은 표현이 섞여 있습니다. 이 매뉴얼에서는 계산 기준을 아래 두 용어로 통일합니다.

통일 용어 의미 실무 해석
근무시간기준 근로제 또는 유형에서 정한 기준 근무시간으로 인정 실제 신청시간보다 기준근무시간이 먼저입니다. 종일 외근이나 종일 출장처럼 본시간을 일정하게 보려는 구조에 가깝습니다.
신청시간기준 신청서에 입력된 시작·종료 시간만큼 인정 시간 단위 출장·외근처럼 신청 시간 범위를 직접 근무시간으로 쓰는 구조입니다.

왜 용어를 바꾸는가

출/퇴근 인정이라는 말은 직원이 실제로 타각해야 하는지, 아니면 계산 기준이 무엇인지가 섞여 보여서 해석이 흔들릴 수 있습니다. 계산 문서에서는 근무시간기준 / 신청시간기준으로 읽는 편이 더 명확합니다.

외근 인정시간 계산

설정 항목 가능 값 인정시간 해석 운영 포인트
인정 단위 종일 / 시간 종일이면 근무시간기준으로 읽고, 시간이면 신청시간기준으로 읽는 구조가 자연스럽습니다. 현재 기획 메모상 외근 인정 단위 설명은 다시 손볼 필요가 있어, 문구보다 실제 계산기준을 함께 표시해야 합니다.
근무시간 자동 인정 사용 / 미사용 사용이면 유형 기준만으로 본시간을 만들고, 미사용이면 실제 타각이나 신청시간과의 관계를 더 봐야 합니다. 자동 인정이 켜져 있으면 단순 승인만으로 본시간이 형성될 수 있으므로, 과대 산정 방지 정책이 필요합니다.
근무시간 계산 기준 근무시간기준 / 신청시간기준 근무시간기준은 근로제 시간 또는 유형 기준시간을, 신청시간기준은 신청 시간대만큼을 반영합니다. 같은 외근이라도 종일형과 시간형을 같은 계산식으로 설명하면 오해가 생깁니다.
연장·야간 인정 사용 / 미사용 사용이면 외근 시간 중 기준을 초과하거나 야간에 해당하는 시간을 별도 버킷으로 분리할 수 있습니다. 본시간 인정과 연장·야간 인정은 같은 스위치가 아니므로 별도로 설명해야 합니다.
휴일 근무 인정 사용 / 미사용 휴일에 발생한 외근을 휴일근무 버킷으로 분리할지 결정합니다. 휴일 캘린더 판정이 먼저이고, 외근 유형은 그 위에서 반영됩니다.

출장 인정시간 계산

설정 항목 가능 값 인정시간 해석 운영 포인트
근무시간 계산 기준 근무시간기준 / 신청시간기준 출장도 외근과 마찬가지로 두 기준으로 갈립니다. 다만 종일 출장은 근무시간기준을 쓰는 경우가 많습니다. 출장 설정 시간 기준을 따로 둘 수 있다면, 근로제 시간과 혼동되지 않게 문서에서 분리해서 보여줘야 합니다.
근무시간 자동 인정 사용 / 미사용 사용이면 승인된 출장 유형만으로 본시간 인정이 가능하고, 미사용이면 추가 실적 또는 신청시간 해석이 필요합니다. 출장은 타각이 없는 경우가 흔하므로 자동 인정 설정이 외근보다 더 큰 영향을 줍니다.
연장·야간 인정 사용 / 미사용 사용이면 출장 중 초과·야간 시간을 따로 인정할 수 있습니다. 자동 인정이 본시간만 만드는지, 연장·야간도 같이 만드는지는 유형 설명에서 분리해야 합니다.
휴일 근무 인정 사용 / 미사용 휴일 출장 시간을 휴일근무로 따로 잡을지 결정합니다. 휴일 여부는 캘린더 기준으로 먼저 확정되어야 합니다.
출퇴근 실적 연계 있음 / 없음 타각이 있으면 실제 시각을 보완자료로 볼 수 있고, 없으면 유형 설정과 신청서가 인정 근거가 됩니다. 출장은 “타각이 없어도 인정 가능”한 대표 케이스이므로, 실적과 보완 규칙을 같이 문서화해야 합니다.

공통 정책

공통 항목 실무 기준 문서화 포인트
본시간 인정 유형 설정에 따라 근무시간기준 또는 신청시간기준 사용 무조건 종일 8시간 같은 식으로 쓰지 말고, 어떤 유형이 어떤 기준을 쓰는지 같이 적습니다.
연장·야간 인정 본시간 인정과 별도 설정 본시간은 인정되지만 연장·야간은 미인정일 수 있으므로 같은 문장으로 묶지 않습니다.
휴일 인정 휴일 캘린더 + 유형 설정 동시 확인 휴일 여부를 먼저 판정한 뒤, 유형이 휴일근무를 허용하는지 확인해야 합니다.
타각의 역할 보완자료 또는 실제 실적 자료 출장·외근은 타각이 없어도 인정될 수 있으나, 타각이 있으면 실제 근무 범위 검토에 활용합니다.

수정 필요 메모

외근 인정 단위 설명

현재 기준 자료에는 외근 인정 단위의 설명을 다시 손봐야 한다는 메모가 있습니다. 따라서 매뉴얼에서는 종일/시간이라는 레이블만 적지 말고, 각 단위가 실제로 근무시간기준인지 신청시간기준인지를 함께 써야 합니다.

용어 통일

기존 자료의 출/퇴근 인정 표현은 계산 기준을 설명할 때는 모호합니다. 이 문서와 관련 보완안에서는 근무시간기준 / 신청시간기준으로 통일해 설명하는 것이 좋습니다.

대표 예시

사례 설정 계산 해석
종일 외근 근무시간 자동 인정 사용, 종일 외근 본시간은 근무시간기준으로 반영합니다. 별도 타각이 없어도 유형 설정만으로 본시간이 잡힐 수 있습니다.
시간 외근 신청 14:00~16:00, 신청시간기준 해당 시간대만 본시간 보완으로 반영합니다. 전일 근무시간을 모두 바꾸는 것은 아닙니다.
종일 출장 + 야간 인정 사용 근무시간 자동 인정 사용, 야간 인정 사용 출장 본시간은 근무시간기준으로 만들고, 야간 해당 구간은 별도 버킷으로 추가 분리할 수 있습니다.
휴일 출장 휴일 근무 인정 미사용 출장 자체는 인정되더라도 휴일근무 버킷은 만들지 않을 수 있습니다. 본시간 인정과 휴일 인정은 같은 설정이 아닙니다.
