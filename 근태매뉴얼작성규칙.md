# 근태매뉴얼 작성 설정

## 1. 문서 목적

이 문서는 `C:\AI IDE\업무자동화\근태관리\근태매뉴얼` 폴더 안에서 작성되는 모든 HTML 매뉴얼의 공통 기준이다.

목적은 다음과 같다.

- 근태관리 전용 매뉴얼의 시각적 톤과 정보구조를 일관되게 유지한다.
- 기존 `html-manual/CLAUDE.md`의 스타일을 참고하되, 근태관리 도메인에 맞는 규칙을 별도로 고정한다.
- 향후 페이지가 추가되더라도 동일한 규칙으로 확장할 수 있게 한다.

## 4. 기본 방향

근태매뉴얼은 `근태 전용 인덱스 + 업무별 상세 페이지` 구조로 작성한다.

원칙은 다음과 같다.

1. 기존 글로벌 인덱스에 바로 편입하지 않는다.
2. 이 폴더 안에서 독립적으로 탐색 가능한 구조를 먼저 완성한다.
3. 화면 설명만 하지 말고 `업무 의미`, `정책 영향`, `마감 영향`까지 함께 설명한다.
4. 피그마 화면과 계산계획 문서를 연결해 실무 매뉴얼로 재구성한다.

## 5. 파일 구조 규칙

### 5.1 필수 구조

- `index.html`
- 업무별 상세 HTML 여러 개
- 이 `CLAUDE.md`

### 5.2 파일명 규칙

- 파일명은 한글 기준으로 작성한다.
- 띄어쓰기는 사용하지 않고 `_`만 사용한다.
- 업무명만 보고도 파일 목적이 드러나야 한다.

예시:

- `근태관리_개요.html`
- `인정근로시간_월마감.html`
- `요청승인_상태흐름.html`

## 6. 인덱스 작성 규칙

`index.html`은 근태관리 매뉴얼의 시작점이다.

반드시 아래 요소를 포함한다.

1. 근태관리 매뉴얼 제목
2. 전체 업무 요약
3. 업무 영역별 진입 카드 또는 탭
4. 페이지별 바로가기 링크
5. 추천 열람 순서
6. 인정근로시간/월마감 같은 핵심 페이지 강조

### 6.1 인덱스 UI 방향

- 기존 `html-manual/CLAUDE.md`의 `Template C` 계열을 참고한다.
- 카드형 또는 탭형 네비게이션을 사용한다.
- 첫 화면에서 전체 구조가 보이도록 한다.
- 너무 긴 본문 설명보다 `탐색 허브` 역할에 집중한다.
- 좌측 메뉴는 가능한 한 `상위 업무군`만 노출한다.
- 상세 문서는 좌측에 모두 늘어놓지 않고, 각 업무군 내부의 `상단 탭` 또는 `서브탭`으로 묶는다.
- 즉 `좌측 = 업무군`, `상단 탭 = 상세 문서`, `본문 = iframe` 구조를 우선 원칙으로 삼는다.

### 6.2 인덱스 링크 규칙

- 모든 상세 페이지는 인덱스에서 1회 이상 링크되어야 한다.
- 상세 페이지 안에도 다시 `index.html`로 돌아오는 링크를 둔다.
- 관련 페이지끼리는 `다음으로 보기` 또는 `연관 업무` 링크를 둔다.

## 7. 상세 HTML 작성 규칙

상세 페이지는 화면 설명서가 아니라 `업무 매뉴얼`이어야 한다.

반드시 아래 흐름으로 작성한다.

1. 페이지 목적
2. 누가 쓰는 화면인지
3. 이 업무가 전체 프로세스에서 차지하는 위치
4. 화면 구성 또는 주요 설정값
5. 처리 흐름
6. 정책/계산 영향
7. 예외 또는 주의사항
8. 관련 페이지 링크

### 7.1 상세 페이지 UI 방향

- 기존 `html-manual/CLAUDE.md`의 `Template A` 계열을 기본으로 사용한다.
- 어두운 배경의 문서형 레이아웃을 우선한다.
- 섹션 제목, 설명 카드, 표, 강조 박스를 적극 활용한다.
- 화면 설명보다 `왜 필요한지`, `어디에 영향을 주는지`를 함께 적는다.

### 7.2 문체 규칙

- 설명형 문장으로 작성한다.
- 짧은 문단과 명확한 소제목을 사용한다.
- 내부 업무 담당자가 읽어도 자연스럽고, 신규 담당자가 읽어도 흐름을 이해할 수 있어야 한다.
- 지나치게 개발자 중심 용어만 쓰지 않는다.

## 12. 용어집 작성 규칙

`용어집.html`은 근태관리 매뉴얼에서 사용되는 전문 용어를 카테고리별로 정리한 참고 문서이다.

### 12.1 용어 추가 기준

매뉴얼에서 아래에 해당하는 용어가 등장하면 용어집에 추가한다.

- 근로기준법·시행령에 근거한 법률 용어 (소정근로시간, 주휴일, 연장근로 등)
- ERP 시스템 고유 용어 (예상 출퇴근 프레임, 인정근로시간, 월마감 등)
- 근무 정책 용어 (간주근로, 포괄임금제, 코어타임, 정산기간 등)
- 처음 근태관리를 접하는 사람이 모를 수 있는 업무 용어

### 12.2 용어 카드 구조

각 용어는 아래 정보를 포함한다.

1. **용어명** — 공식 명칭 + 별칭이 있으면 괄호로 함께 표기
2. **정의** — 1~3문장. 무엇인지, 왜 필요한지, 어디에 영향을 주는지
3. **관련 문서** — 해당 용어가 주로 등장하는 매뉴얼 페이지명
4. **data-term 속성** — 검색 필터용. 용어명 + 별칭 + 유사어를 공백으로 연결

### 12.3 카테고리 분류

용어는 아래 10개 카테고리 중 하나에 배치한다.

| 카테고리 | 설명 |
|----------|------|
| 근로제 유형 | 표준·시차·선택·스케줄근로제 등 |
| 법정 시간·근로 기준 | 소정근로시간, 연장·야간·휴일근로, 휴게시간 등 |
| 근태 계산 핵심 | 인정근로시간, 예상 출퇴근 프레임, 월마감, 타각 등 |
| 근무 정책 | 간주근로, 포괄임금제, 코어타임, 유예시간 등 |
| 근무패턴 | 근무패턴, 반복설정, 정상근로인정 등 |
| 출퇴근 방식 | GPS/WIFI/WEB IP 기반 출퇴근, 자동등록 등 |
| 휴가 | 연차, 반차, 시간차, 유효기간, 미차감 등 |
| 출장·외근 | 출장유형, 근무인정시간, 간주근무 등 |
| 근태 요청 | 수정요청, 사후신청, 일괄확정 등 |
| 시스템·운영 | 배포, 재마감, 정합성체크, 급여연동 등 |

### 12.4 유지 원칙

- 새 매뉴얼 페이지를 작성할 때 신규 전문 용어가 등장하면 용어집에도 함께 추가한다.
- 용어 정의가 변경되면 용어집의 해당 카드도 갱신한다.
- 중복 용어를 만들지 않는다. 추가 전 기존 항목을 먼저 확인한다.
- index.html의 Appendix 섹션에서 용어집으로 진입할 수 있어야 한다.
