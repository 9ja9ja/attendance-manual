인정근로시간은 일별 분류 데이터를 만든 뒤 월 근태 마감으로 확정합니다

이 서비스는 근로제 → 근무패턴 → 예상 출퇴근 시간 → 직원 출/퇴근 저장 → 집계마감 기준데이터 생성 구조를 전제로 설계되어 있습니다. 따라서 월마감은 출퇴근기록을 단순 합산하는 절차가 아니라, 정상 출퇴근, 연장 인정, 근태 이상, 휴일 근무로 분류된 일별 기준데이터를 공식 결과로 확정하는 단계입니다.

핵심 분류: 정상 / 연장 / 이상 / 휴일 최종 산출: 월마감 확정 결과
목차
월마감 계산의 기본 원칙 전체 흐름 입력 원천과 우선순위 표준/시차 근로제와 선택 근로제의 차이 일별 판정 상세 단계 집계마감 기준데이터 분류 일별 산식과 결과 컬럼 월 집계와 급여 연동 마감 전 체크리스트 예외 처리와 보정 로그 현재 구조에서의 운영 기준 함께 보면 좋은 문서

월마감 계산의 기본 원칙

최종안 핵심 요약
  • 타각만으로 확정하지 않고, 근로제·근무패턴·승인 완료 데이터·유형 기준정보를 함께 봅니다.
  • 예상 출퇴근 시간은 실제 출퇴근과 별개로 존재하는 비교 프레임입니다.
  • 총 인정근로시간과 연장·야간·휴일 시간은 속성 분해 방식으로 별도 관리합니다.
  • 일별 분류 데이터를 먼저 만들고, 월마감은 그 데이터를 집계·검토·확정하는 단계입니다.
  • 마감 후 결과는 급여 반영용 확정 자료로 사용합니다.

전체 흐름

1. 근로제 등록

근무 유형과 기본 소정 조건을 정의합니다.

2. 근로제 유형 설정

표준/시차 근로제와 선택 근로제 흐름이 분기됩니다.

3. 정보/정책 설정

근로제 정보와 정책 설정이 계산 기준값을 만듭니다.

4. 근무 패턴 추가

요일별 근무 프레임과 반복 기준을 구체화합니다.

5. 예상 출퇴근 수립

직원별 일자 기준 예상 출퇴근 시간 프레임을 생성합니다.

6. 직원 출퇴근 저장

실제 출근/퇴근 실적을 저장합니다.

7. 기준데이터 생성

정상 출퇴근, 연장 인정, 근태 이상, 휴일 근무로 분류합니다.

8. 월 집계와 확정

관리자 검토 후 월마감 결과와 급여 반영 자료를 확정합니다.

9. 급여 반영

확정 결과를 급여 계산의 공식 기준으로 사용합니다.

입력 원천과 우선순위

우선순위 입력 원천 매뉴얼 해석
1개인별 확정 스케줄 / 캘린더 / 근로제근무일인지 휴일인지, 소정근로시간이 얼마인지, 어떤 유형의 근로제인지 판정하는 최상위 기준입니다.
2근무패턴 / 예상 출퇴근 프레임정상 범위와 비교 기준을 만듭니다. 별도 단계이므로 독립 입력값으로 보는 것이 맞습니다.
3승인 완료된 근태 기안휴가, 출장, 외근, 연장, 휴일근무, 수정 요청을 공식 판정 데이터로 반영합니다.
4실제 출퇴근 / 휴게 실적기본 실적입니다. 하지만 승인 데이터와 유형 설정 없이 단독으로 최종 확정되지는 않습니다.
5자동 규칙휴게 자동 차감, 절사, 지각/조퇴 기준 같은 공통 규칙을 적용합니다.
6관리자 보정 / 마감 확정누락 타각, 충돌 케이스, 소급 수정 등 예외를 최종 확정합니다.

표준/시차 근로제와 선택 근로제의 차이

표준 / 시차 근로제

일 단위 예상 출퇴근 프레임이 비교적 명확합니다. 따라서 정상 출퇴근, 지각, 조퇴, 연장 인정 여부를 하루 단위로 직관적으로 판정할 수 있습니다.

선택 근로제

별도 분기로 처리되며, 주 단위 또는 월 단위 정산이 강조됩니다. 즉 예상 출퇴근 프레임은 존재하더라도, 최종 초과근무 판정은 정산기간 기준으로 재판정할 필요가 있습니다.

선택 근로제는 예상 출퇴근 시간 수립과 정산기간 단위 초과근무 판정이 모두 필요합니다. 따라서 일반근로제와 같은 일 단위 계산만으로 설명하면 빠지는 내용이 생깁니다.

일별 판정 상세 단계

단계 설명 실무 포인트
0단계 대상 확정마감 대상 월, 직원, 사업장, 조직을 확정합니다.월중 입퇴사자와 겸직자, 조직 이동자를 함께 고려해야 합니다.
1단계 기준 자료 확정캘린더, 근로제, 휴가유형, 출장유형, 정책값을 마감 기준 자료로 고정합니다.마감 후 기준값이 바뀌더라도 기존 결과 설명이 가능해야 하므로 기준 자료 확정이 필요합니다.
2단계 정합성 체크미승인 요청, 누락 타각, 미확정 스케줄, 미완료 보정 여부를 확인합니다.이 단계는 마감 가능 여부를 결정하는 품질 게이트입니다.
3단계 기본 근무일 판정근무일, 유급휴일, 무급휴무일, 대체휴일, 휴일을 판정합니다.휴일 판정이 틀리면 이후 휴일근로 분류가 전부 흔들립니다.
4단계 소정근로 프레임 생성근로제와 캘린더로 기본 소정근로 구간을 만듭니다.이 단계는 "원래 어떻게 일하기로 되어 있었는가"를 정의합니다.
4-A단계 예상 출퇴근 수립근무패턴을 반영해 예상 출근/퇴근 프레임을 직원별로 생성합니다.독립 단계이며, 정상/이상 판정의 핵심 기준입니다.
5단계 원본 실적구간 생성실제 출퇴근과 휴게 실적으로 기본 실적구간을 만듭니다.타각은 기본 실적이지만 최종 인정근로시간과 동일하지는 않습니다.
6~8단계 승인/휴가/출장 반영승인 완료된 요청과 휴가/출장 유형을 반영해 판정대상시간을 보정합니다.출장유형의 출퇴근 인정, 근무 인정 시간, 휴게 인정 시간이 직접 개입합니다.
9단계 판정대상시간 생성실적구간과 승인된 간주근무 구간을 합쳐 실제 판정대상시간을 만듭니다.이 시점부터 단순 타각이 아닌 "회사가 인정할 후보 시간"이 됩니다.
10단계 휴게 차감실휴게 또는 자동 휴게 규칙을 반영합니다.간주근무의 휴게 인정 규칙도 함께 봐야 합니다.
11단계 인정근로시간 산출판정대상시간에서 제외시간과 휴게시간을 차감해 일별 인정근로시간을 만듭니다.관리자 확정 보정이 있다면 이 단계에서 최종 반영합니다.
12단계 시간 속성 분해소정, 연장, 야간, 휴일 시간으로 분해합니다.총 인정근로시간과 수당용 시간 속성을 반드시 분리합니다.
13단계 이상 판정누락 타각, 지각, 조퇴, 승인 없는 초과근무 같은 상태를 생성합니다.근태 이상은 별도 결과 상태로 관리됩니다.
13-A단계 기준데이터 분류정상 출퇴근, 연장 인정, 근태 이상, 휴일 근무로 분류합니다.월마감은 이 분류 데이터를 집계하는 구조로 설명하는 것이 가장 정확합니다.
14~17단계 주간 정산 / 월 집계 / 보정 / 확정유연근무 정산, 월합계 계산, 관리자 검토, 확정 결과 저장을 수행합니다.이후 급여는 원본이 아니라 확정 결과를 읽어야 합니다.

집계마감 기준데이터 분류

집계마감 기준데이터는 네 가지로 분류됩니다. 이 네 분류는 단순 요약이 아니라, 마감 직전 일별 결과를 어떻게 저장하고 검토할지를 보여주는 핵심 개념입니다.

정상 출퇴근

예상 출퇴근 프레임 안에서 정상적으로 근무가 수행된 일자입니다. 일별 기본 근태의 기준 그룹입니다.

연장 인정

소정근로를 초과했으며 정책상 인정 가능한 시간입니다. 승인 여부와 근로제 정책이 함께 반영됩니다.

근태 이상

누락 타각, 지각, 조퇴, 승인 없는 초과근무, 비교 프레임 불일치 등 운영 확인이 필요한 상태입니다.

휴일 근무

휴일 판정일에 근무로 인정된 시간입니다. 출장/외근 유형에서 휴일 포함 여부도 함께 영향을 줍니다.

일별 산식과 결과 컬럼

일별 기본 산식

판정대상시간 = 실제 출퇴근 구간 + 승인된 간주근무 구간

인정근로시간 = 판정대상시간 - 휴게시간 - 제외시간 ± 관리자 확정보정

일별 결과 컬럼

인정근로시간, 소정근로시간, 연장근로시간, 야간근로시간, 휴일근로시간, 유급휴가시간, 무급휴가시간, 근태이상여부, 보정사유, 근거문서번호를 분리 저장하는 방향이 적합합니다.

야간은 총시간을 한 번 더 더하는 개념이 아니라, 이미 인정된 시간의 속성입니다. 따라서 총 인정근로시간과 가산수당용 시간을 분리 저장해야 급여 계산이 깔끔합니다.

월 집계와 급여 연동

구분 설명 매뉴얼 포인트
월 집계 결과월 총 인정근로시간, 월 소정근로시간, 월 연장·야간·휴일근로시간, 월 유급/무급휴가 시간을 집계합니다.임금명세서와 급여 수당 계산에 바로 연결됩니다.
관리자 검토 및 보정분류 결과를 보고 이상 상태나 예외 케이스를 마지막으로 정리합니다.근태 요청 관리 화면과 함께 마감 직전 운영 작업이 수행됩니다.
확정 결과월마감 완료 시 급여 이관용 확정 자료를 생성합니다.급여는 원본 타각이 아니라 확정 결과를 읽어야 합니다.
재마감 절차마감 후 수정이 필요하면 해제 후 재계산하고 다시 확정합니다.소급 변경 정책을 별도로 운영해야 합니다.

마감 전 체크리스트

  • 미승인 근태 요청이 남아 있지 않은지 확인합니다.
  • 누락 타각과 근태 이상 상태가 정리되었는지 확인합니다.
  • 근로제와 근무패턴, 캘린더 기준 자료가 생성 가능한지 확인합니다.
  • 휴가유형, 출장유형 같은 유형 기준정보 변경이 과거 기간에 영향을 주는지 확인합니다.
  • 선택근무제 대상자는 정산기간 단위 판정이 별도로 반영되었는지 확인합니다.

예외 처리와 보정 로그

자주 발생하는 예외

승인 없는 연장, 누락 타각, 자정 넘김, 휴게 미입력, 출장인데 타각 없음, 휴가와 출퇴근 기록 동시 존재 같은 케이스가 대표적입니다.

로그에 남겨야 할 항목

적용 근로제, 예상 출퇴근 프레임, 실제 타각, 승인 문서번호, 유형 기준값, 관리자 보정자, 보정 사유를 남겨야 결과 설명과 감사가 가능합니다.

현재 구조에서의 운영 기준

현재 화면 구성상 근로제, 근무패턴, 근태 요청, 직원 근태, 휴가유형, 출장유형, 출퇴근 인정 근로 시간 계산 흐름이 모두 존재합니다. 따라서 현재 구조에서도 충분히 일별 분류 데이터 생성 → 월 집계 → 관리자 검토 → 급여 반영 자료 확정 구조로 운영 기준을 설명할 수 있습니다. 이 매뉴얼은 현재 화면 구성을 기준으로 설명하고 있으며, 세부 산식은 별도 계산계획 문서와 일치하게 유지합니다.
