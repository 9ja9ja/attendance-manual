인정근로시간은 입력원천의 우선순위와 충돌 규칙까지 정의해야 완성됩니다

이 문서는 기존 인정근로시간·월마감 및 인정근로시간 통합 안내 문서를 보완하기 위한 상세 매뉴얼입니다. 기존 문서가 잘 잡아놓은 전체 흐름은 유지하되, 실제 시스템 계산에서 가장 중요한 입력원천 우선순위, 충돌 시 처리 원칙, 원시실적과 승인데이터의 관계, 무엇을 대체하고 무엇을 보완하는지를 더 구체적으로 정리합니다. 통합로직 문서에 정상 반영된 일반 흐름은 반복하지 않고, 그 흐름만으로는 부족한 예외, 충돌, 미확정 규칙만 추가로 다룹니다.

실무 판정 우선순위

실무적으로 바로 적용하기 좋은 판정 순서는 아래와 같습니다. 핵심은 무조건 타각 우선도 아니고, 무조건 기안 우선도 아니라는 점입니다. 먼저 그날의 근로의무와 날짜 성격을 정하고, 그 다음 승인된 기안으로 근로의무 면제 또는 간주근무를 반영한 뒤, 실제 타각으로 충돌 여부와 실제 근로 흔적을 확인하는 방식이 가장 안정적입니다.

실무 순위 입력원천 주요 역할 운영 원칙
1 근무패턴 / 회사 캘린더 / 개인 스케줄 소정근로일 여부, 휴일 여부, 소정시간, 휴게 배치, 예상 프레임 결정 이 기준이 없으면 연차, 지각, 휴일근로, 결근을 판정할 수 없습니다.
2 승인 완료된 근태기안 연차, 반차, 시간연차, 출장, 외근, 재택, 휴일근무, 연장근무, 대체휴무 반영 근로의무 면제 또는 간주근무 성격의 기안은 원칙적으로 타각보다 먼저 해석합니다.
3 타각기록 실제 출근/퇴근, 외출/복귀, 모바일/GPS/PC 기록 등 실제 근무 흔적 확인 최상위 원천은 아니지만, 충돌 탐지와 실제 근로 제공 확인에 가장 중요합니다.
4 자동 규칙 휴게 자동 차감, 지각/조퇴 기준, 절사 규칙, 연차 시간 환산 적용 앞단 데이터를 해석하는 보조 규칙이지, 앞단 원천을 새로 만들지는 않습니다.
5 관리자 보정 오타각 정정, 출장 미타각 보정, 연차 취소/변경, 근로 제공 사실 확정 충돌, 오류, 미확정 상태를 최종 확정하는 단계입니다.
6 마감 확정값 급여와 정산으로 넘길 최종 결과 유지 재마감 전에는 변하지 않는 최종 결과로 관리합니다.
정리하면 근무패턴·캘린더 > 승인 완료된 근태기안 > 타각기록 > 자동 규칙 > 관리자 보정 > 마감 확정 순으로 보는 것이 가장 실무적입니다.
특히 연차·반차·공가처럼 근로의무 면제 성격의 기안과, 출장·외근·재택처럼 근무 간주 또는 보완 성격의 기안은 같은 승인 데이터라도 처리 방식이 다릅니다. 전자는 시간을 비근로 상태로 대체하고, 후자는 실제 부족분을 보완하거나 근무로 간주합니다.

대표 충돌 시나리오

아래 시나리오는 실제 ERP 규칙으로 바로 옮겨 쓰기 좋은 대표 케이스입니다. 충돌이 발생하는 순간 무리하게 자동확정하지 않고, 무엇이 정상이고 무엇이 경고, 충돌, 오류, 미확정인지 먼저 나누는 것이 중요합니다.

여기서 말하는 짧은 타각과 장시간 타각은 회사 규정으로 기준을 두는 것이 좋습니다. 예를 들어 30분 미만 체류는 경고, 30분 이상 체류는 충돌처럼 두면 운영 기준을 만들기 쉽습니다. 다만 이 기준은 업종과 출퇴근 운영방식에 따라 조정할 수 있어야 합니다.
시나리오 1차 자동처리 최종 원칙
종일 연차 승인 + 타각 없음 연차 확정 정상입니다. 결근이 아니라 연차 사용으로 보고 소정근로시간만큼 차감합니다.
종일 연차 승인 + 짧은 타각 연차 유지 + 경고 오타각 또는 잠시 방문 가능성이 있으므로 경고만 남기고 연차를 유지할 수 있습니다.
종일 연차 승인 + 장시간 타각 충돌 상태 연차 취소 누락, 오타각, 실제 근로 제공 중 무엇인지 관리자 확인이 필요합니다.
반차 승인 + 반차 시간대 타각 없음 반차 확정 정상입니다. 반차 시간은 비근로 시간대로 대체하고 나머지 시간만 근무 판정합니다.
반차 승인 + 반차 시간대 근무 타각 충돌 상태 반차 취소, 시간단위 휴가 전환, 실제 근무 인정 중 하나로 정리해야 합니다.
정상근무일 + 타각 없음 + 기안 없음 미확정 즉시 결근 확정하지 말고 마감 전까지 보정 대기 후, 미보정이면 결근으로 확정합니다.
출장 승인 + 타각 없음 출장근무 인정 정상입니다. 소정근로시간 인정 또는 출장근무 규칙을 적용합니다.
휴일 + 타각 + 휴일근무 승인 없음 충돌 상태 단순 체류인지, 실제 지시 근무인지, 승인 누락인지 확인한 뒤 반영합니다.
휴무일 + 연차 승인 오류 원래 근로의무가 없는 날에는 연차 차감이 성립하지 않으므로 오류로 처리하는 편이 안전합니다.
연차 시간대와 연장근무 시간대 중복 오류 시간대 중복은 자동으로 성립시키지 말고, 기안 변경 또는 시간대 재조정이 필요합니다.
특히 종일 연차 + 실제 타각, 반차 + 반차 시간대 타각, 휴일 + 장시간 타각은 자동확정하면 안 되는 대표 사례입니다. 실제 근로 제공이 있었을 가능성이 크기 때문입니다.

상태값 체계와 자동확정 기준

충돌 규칙을 운영에 연결하려면 단순히 이상 여부만 두기보다, 상태값 체계를 명확히 가져가는 것이 좋습니다. 아래 체계는 월마감 전 정리 대상과 급여 반영 가능 여부를 분명하게 나누는 데 유용합니다.

상태값 의미 마감 가능 여부
정상 승인기안과 타각이 모순되지 않고 자동확정 가능한 상태 가능
경고 운영상 확인은 필요하지만 자동확정은 가능한 상태 가능
충돌 둘 이상의 원천이 동시에 성립해 자동으로 결정할 수 없는 상태 불가
오류 원칙적으로 동시에 성립하면 안 되는 조합이 발생한 상태 불가
미확정 증빙이나 보정이 아직 부족해 결론을 내리지 못한 상태 불가

자동확정 가능한 것

  • 승인기안과 타각이 모순되지 않는 경우
  • 종일 연차 + 무타각
  • 출장 승인 + 무타각
  • 반차 + 타각이 반차 외 시간대에만 존재하는 경우

자동확정하면 안 되는 것

  • 연차/반차 시간대와 타각이 겹치는 경우
  • 휴일인데 장시간 타각이 있는 경우
  • 휴가기안과 출장기안이 같은 시간대에 겹치는 경우
  • 휴무일에 연차 차감이 발생하는 경우
운영 원칙은 정상 / 경고만 자동확정하고, 충돌 / 오류 / 미확정은 관리자 보정 이력 없이는 월마감과 급여 반영이 불가능하도록 막는 방식이 가장 실무적입니다.

문서 목적과 요약

기존 문서는 기준정보 → 실적수집 → 승인반영 → 일별판정 → 월마감확정 → 급여반영이라는 큰 흐름을 잘 잡고 있습니다. 다만 실제 계산규칙으로 내리려면, 각 입력원천이 비교 기준인지, 대체값인지, 보완값인지, 교정값인지, 최종확정값인지까지 나누어야 합니다.

즉, 지금 필요한 보완은 흐름 자체를 뒤집는 것이 아닙니다. 오히려 기존 흐름을 유지하면서도 각 단계가 무엇을 하는지 더 엄격하게 정의하는 것입니다. 특히 판정대상시간 = 실제 타각 + 승인 간주근무 같은 표현은 방향은 맞지만, 그대로 쓰면 중복 산입이나 충돌 오판정이 생길 수 있습니다.

기존 문서의 강점

예상 프레임을 먼저 만든다

근로제와 근무패턴을 먼저 보고, 그 다음 실제 출퇴근을 비교한다는 구조가 분명합니다. 이 방향은 근태 시스템 설계에서 매우 안정적입니다.

일별 분류 후 월마감

하루하루를 먼저 정상, 연장, 이상, 휴일로 분류하고 그 결과를 월마감에서 확정한다는 구조가 일관됩니다.

가산시간 분리 인식

총 인정근로시간과 연장·야간·휴일 시간을 분리해 관리해야 한다는 점을 이미 잘 짚고 있습니다.

기존 문서에서 보완이 필요한 부분

보완 필요 항목 현재 상태 왜 문제인지
입력원천 우선순위 두 문서에 표현은 있으나 순서와 의미가 완전히 같지는 않습니다. 기획서와 구현기준이 달라질 수 있고, 테스트 케이스 정의도 흔들릴 수 있습니다.
실적과 승인데이터의 관계 승인 완료 데이터가 중요하다고 설명하지만, 실제 실적을 덮는지 보완하는지까지는 구분이 약합니다. 휴가 + 실제 타각, 출장 자동인정 + 실제 타각 같은 케이스에서 중복 계산이 생길 수 있습니다.
정정 승인과 원시타각 관계 수정 요청이 반영된다는 설명은 있지만, 원시값을 교체하는지 누적하는지 명시가 부족합니다. 정정 데이터가 보정인지 누적인지 불분명하면 최종 시각이 흔들립니다.
법정 시간과 회사 승인시간의 분리 연장 인정, 승인 없는 연장, 급여 반영이 같은 그룹 안에서 보이는 표현이 있습니다. 법정 근로시간 발생과 회사 승인 여부는 다른 층위인데, 이를 분리하지 않으면 법령 설명이 약해집니다.

원래 문서 대비 무엇이 바뀌어야 하는가

기존 문서 표현 보완 후 권장 표현 변경 이유
승인 완료된 요청을 공식 판정 데이터로 반영 승인 완료된 요청은 실적을 대체하거나 보완하거나 교정하는 거래데이터로 반영 승인 데이터의 역할이 모두 같지 않기 때문입니다.
판정대상시간 = 실제 출퇴근 구간 + 승인된 간주근무 구간 교정실적구간 + 보완근무구간 - 비근로대체구간 단순 합산으로 표현하면 이중 산입 위험이 큽니다.
승인 없는 연장 = 이상 상태 승인 없는 연장은 운영상 이상 상태이되, 법정 발생시간과 회사 승인시간은 분리 관리 내부 승인 규칙과 법정 시간 발생은 같은 개념이 아닙니다.
휴가/출장 반영 휴가는 근무의무 대체, 출장은 부족구간 보완 또는 간주근무 생성 휴가와 출장은 데이터 성격이 다르므로 같은 레벨로만 처리하면 안 됩니다.
근무패턴은 예상 프레임 생성 기준 근무패턴은 기준근무시간을 가감하고, 그 결과로 예상 프레임을 만드는 기준 FigJam 기준으로 근무패턴은 비교 프레임만이 아니라 기준근무시간 산정에도 영향을 줍니다.
출장/외근은 타각 부족분 보완 출장/외근은 출퇴근 인정 여부에 따라 근무 시간 기준 또는 신청 시간 기준으로 반영 모든 출장·외근을 같은 방식으로 계산하면 유형별 정책 차이를 설명할 수 없습니다.
연장 인정은 하나의 처리 방식 연장 자동 인정은 본시간 반영 경로, 전자결재 인정은 연장시간 반영 경로로 구분 자동 인정과 승인 인정은 같은 연장이라도 반영 경로와 정산 의미가 다릅니다.
관리자 보정 반영 관리자 보정은 최후수단이며, 반드시 보정사유와 근거문서를 남김 마감 후 결과 설명과 감사 대응을 위해 보정 이력이 필수입니다.
기존 두 문서는 "흐름"을 설명하는 문서로는 충분히 좋습니다. 이번 보완 문서는 여기에 계산의 질서를 더하는 문서라고 보면 됩니다. 즉, 어떤 데이터가 먼저인지보다 더 중요한 것은 그 데이터가 무엇을 바꾸는 데이터인지입니다.

권장 계산모델

인정근로시간은 한 줄 산식으로 끝나는 개념이 아니라, 여러 층을 거쳐 만들어지는 결과값입니다. 가장 안전한 구조는 아래 5개 레이어를 나누는 방식입니다.

1. 기준정보

근로제, 캘린더, 근무패턴, 휴가유형, 출장유형

2. 원시실적

원시 출근/퇴근, 외출/복귀, 실제 휴게

3. 승인거래

휴가 승인, 출장 승인, 출퇴근 정정 승인, 연장 승인

4. 자동규칙

자동 휴게, 절사, 자정 넘김 분리, 지각/조퇴 기준

5. 관리자확정

보정, 소급정정, 월마감 확정, 재마감

6. 급여반영

확정 결과를 급여 기준자료로 이관

교정실적구간 = 원시실적구간 + 승인된 출퇴근정정 반영

근무의무구간 = 근로제 + 캘린더 + 근무패턴 기준 구간

상태대체구간 = 승인된 휴가/휴무 등으로 근무의무를 대체하는 구간

보완근무구간 = 출장/외근 간주근무 등으로 실제 부족분을 채우는 구간

판정대상시간 = 교정실적구간 + 보완근무구간 - 비근로대체구간

인정근로시간 = 판정대상시간 - 휴게시간 - 제외시간 ± 관리자보정

입력원천 우선순위

이 섹션은 엔진 내부에서 입력원천을 어떤 성격으로 다룰지를 설명하는 표입니다. 앞에서 정리한 실무 판정 우선순위가 운영자가 날짜 성격과 승인 여부를 어떤 순서로 판단할지 보여주는 표라면, 아래 표는 각 데이터가 기준, 사실, 대체, 보완, 교정, 확정 중 무엇으로 작동하는지 설명하는 내부 구조 안내입니다.
우선순위 입력원천 역할 비고
1 개인별 근로제 / 캘린더 / 근무패턴 근무일 여부, 휴일 여부, 소정근로시간, 예상 프레임을 결정하는 기준값 이 단계는 실적을 대체하지 않고, 판정 기준을 만듭니다.
2 원시 출퇴근 / 외출 / 실제 휴게 사실 데이터 이 단계는 존재한 사실을 저장합니다. 아직 인정 여부는 확정하지 않습니다.
3 승인된 출퇴근 수정 / 근로제 변경 / 휴가 / 출장 / 연장 요청 원시실적을 교정하거나, 근무의무를 대체하거나, 부족구간을 보완하는 거래 데이터 모든 승인 데이터가 동일한 방식으로 작동하지 않으므로 역할 구분이 필요합니다.
4 자동 규칙 앞 단계 결과를 해석하는 계산 보조 규칙 자동규칙은 원시실적과 승인을 뒤엎기보다 해석하는 성격이어야 합니다.
5 관리자 보정 / 월마감 확정 예외와 충돌을 최종 확정하는 단계 반드시 로그와 근거문서가 있어야 합니다.
여기서 중요한 것은 우선순위가 높으면 무조건 덮어쓴다가 아니라, 각 원천이 무엇을 담당하는 데이터인지를 먼저 나누는 것입니다. 근무패턴은 비교 기준이고, 휴가는 상태 대체이며, 출퇴근 수정 승인은 원시값 교정입니다.

입력원천이 갖는 역할 구분

구분 설명 대표 예시
기준 정상/이상/휴일 판정의 비교 프레임이 되는 데이터 근로제, 캘린더, 근무패턴, 예상 출퇴근 프레임
사실 실제로 발생한 근무 흔적을 담는 데이터 원시 타각, 외출/복귀, 실제 휴게
대체 원래 근무해야 할 시간을 다른 상태로 바꾸는 데이터 전일 휴가, 반차, 무급휴무, 휴직
보완 실제 실적이 부족하거나 산정이 어려울 때 일부 구간을 채우는 데이터 출장 자동인정, 외근 간주근무
교정 원시값을 정정하여 최종 기준값으로 바꾸는 데이터 출근/퇴근 수정 승인
확정 예외 판단을 끝내고 결과를 굳히는 데이터 관리자 보정, 월마감 확정

충돌 규칙 상세

아래 규칙은 실제 구현과 테스트케이스를 만들 때 바로 사용할 수 있는 수준으로 정리한 권장안입니다.

1. 근로제/캘린더 vs 실제 타각

근로제와 캘린더는 해당 날짜가 근무일인지, 휴일인지, 소정근로시간이 얼마인지를 정합니다. 실제 타각은 근무의 흔적일 뿐이므로, 타각만으로 휴일이 평일로 바뀌지는 않습니다. 따라서 날짜 성격은 기준정보가 먼저 결정하고, 타각은 그 위에서 근무 발생 여부를 판단해야 합니다.

2. 근무패턴/예상 프레임 vs 실제 타각

예상 프레임은 대체값이 아니라 비교 기준입니다. 실제 타각이 예상보다 늦으면 지각 또는 예외, 빠르면 조퇴 또는 예외, 길면 연장 또는 초과근무 후보로 분류합니다. 예상 프레임과 다르다고 바로 버리는 구조는 피해야 합니다.

3. 휴가 승인 vs 실제 타각

상황 권장 처리 설명
전일 휴가 승인 + 전일 실제 근무 타각 충돌건으로 분리 휴가 상태와 실제 근무가 동시에 존재하므로 자동합산하지 말고 관리자 확인 대상로 보냅니다.
오전 반차 승인 + 오후 실제 근무 휴가 시간대는 비근로 대체, 나머지 시간대만 근무판정 반차/시간차는 부분 대체 규칙으로 봐야 합니다.
시간차 승인 + 전체 타각 존재 승인 시간대를 근무의무 대체구간으로 우선 반영 실제 체류가 있어도 승인 휴가시간은 먼저 비근로상태로 전환해야 합니다.

4. 출장/외근 자동인정 vs 실제 타각

상황 권장 처리 설명
실제 타각 없음 승인된 출장유형의 간주근무 사용 이 경우 출장유형이 부족 구간을 채우는 보완값으로 작동합니다.
실제 타각 일부 존재 실제 타각 우선 + 부족 구간만 간주근무 보완 실제 있는 구간과 자동인정 구간을 모두 더하지 않도록 주의해야 합니다.
실제 타각 충분히 존재 실제 타각 우선, 출장은 상태·사유 중심 유지 자동인정 시간을 다시 더하면 이중 산입 위험이 큽니다.

5. 출퇴근 수정 승인 vs 원시 타각

출퇴근 수정 승인은 원시 타각을 교정하는 데이터입니다. 즉 원시 타각 + 수정 타각이 아니라, 정정 후 기준 타각을 따로 만들어야 합니다. 저장 구조도 raw_clock_in/out과 corrected_clock_in/out를 분리하는 방식이 가장 안전합니다.

6. 연장근무 승인 vs 실제 초과근무

연장 승인 자체가 근로시간을 생성하면 안 됩니다. 연장 승인은 회사가 초과근무를 어떻게 인정할지의 기준이고, 실제 시간은 타각 또는 별도 증빙에서 와야 합니다.
구분 관리 항목 설명
법정 발생시간 실제 초과근무 기준 실제 근무가 있었는지에 따라 발생하는 시간입니다.
회사 승인시간 승인된 초과근무 기준 내부 승인 정책상 인정된 시간입니다.
급여 반영시간 정산·보상 기준 정책에 따라 법정 발생시간과 같을 수도, 회사 승인시간과 같을 수도 있습니다.

7. 실제 휴게 vs 자동 휴게

실제 휴게 입력이 있으면 실제 휴게를 우선 사용하고, 실제 휴게가 없을 때만 규정 휴게를 보조적으로 적용하는 구조가 좋습니다. 출장·외근 간주휴게는 다시 그보다 뒤에 와야 하며, 세 가지 휴게를 동시에 누적하지 않도록 해야 합니다.

8. 관리자 보정 vs 앞단 데이터

관리자 보정은 최후 수단입니다. 앞단 데이터를 무시할 수는 있지만, 반드시 보정사유, 보정자, 보정시각, 근거문서, 원래값, 변경값을 남겨야 합니다. 또한 월마감 후 보정은 재마감 절차와 연결되어야 합니다.

9. 근무패턴 기반 기준시간 계산

FigJam 기준으로 근무패턴은 단순히 예상 출퇴근 프레임을 만드는 데이터가 아닙니다. 먼저 패턴 사용 여부를 판단하고, 패턴이 있는 경우에는 추가 근무인지 단축 근무인지에 따라 해당 일자의 기준근무시간을 다시 계산해야 합니다.

패턴 상태 권장 처리 설명
패턴 미사용 근로제 시간 기준 사용 해당 일자의 기준근무시간은 기본 근로제에서 정한 소정근로시간으로 봅니다.
패턴 사용 + 추가 근무 근로제 시간 + 패턴 근무 시간 추가 근무 예정일로 보고 기준근무시간을 늘린 뒤 예상 프레임을 생성합니다.
패턴 사용 + 단축 근무 근로제 시간 - 패턴 근무 시간 단축 근무 예정일로 보고 기준근무시간을 줄인 뒤 예상 프레임을 생성합니다.
따라서 근무패턴과 실제 타각의 충돌을 볼 때는 예상 프레임만 비교하지 말고, 그 패턴이 해당 일자의 기준근무시간 자체를 바꿨는지까지 먼저 확인해야 합니다.

10. 연장근무 자동 인정 vs 전자결재 인정

연장근무는 한 가지 경로로만 반영되지 않을 수 있습니다. FigJam 기준으로는 연장근무 자동 인정과 연장근무 전자결재 인정이 별도 경로로 표현되어 있으므로, 운영 규칙도 이를 분리해 두는 편이 안전합니다.

구분 권장 반영 경로 설명
연장근무 자동 인정 근무 시간 반영 경로에서 처리 유형 설정이나 기준시간 비교로 자동 인정되는 초과 구간은 본시간 반영 경로에서 함께 해석하는 편이 자연스럽습니다.
연장근무 전자결재 인정 연장 근무 시간 반영 경로에서 처리 승인된 연장시간은 본시간과 분리된 연장시간 버킷으로 관리하는 편이 정산과 설명에 유리합니다.
두 경로 모두 최종적으로는 연장근무와 연결되지만, 자동 인정은 기준시간 비교 결과에 가깝고, 전자결재 인정은 승인 거래에 가깝다는 차이를 남겨야 합니다.

11. 출장/외근의 근무 시간 기준 vs 신청 시간 기준

출장·외근은 항상 같은 방식으로 본시간을 만들지 않습니다. 유형 설정과 출퇴근 인정 여부에 따라 근무 시간 기준 또는 신청 시간 기준으로 갈라지는 구조를 따로 두는 것이 좋습니다.

분기 기준 권장 처리 설명
출/퇴근 인정 안 함 근무 시간 기준 사용 근로제 기준시간 또는 유형 설정 기준으로 본시간을 반영합니다.
출/퇴근 인정 신청 시간 기준 사용 승인된 출장·외근 신청 시간대를 기준으로 근무시간을 반영합니다.
연장·야간 자동 인정 유형 정책에 따라 본시간 또는 연장시간 가산 자동 인정 설정이 있으면 해당 시간 속성을 같이 반영합니다.
연장·야간 미인정 본시간만 반영 출장·외근 자체는 인정하되, 초과·야간 가산 속성은 붙이지 않는 방식입니다.

12. 근태 이상 상태의 본시간 반영 기준

근태 이상은 단순 오류 메시지가 아니라, 본시간을 바로 반영할지 보류할지를 결정해야 하는 상태입니다. 따라서 이상 상태를 발견하는 것과 본시간 반영 여부를 확정하는 것을 같은 단계로 묶지 않는 편이 좋습니다.

상황 권장 처리 설명
경미한 이상 경고 상태 + 본시간 반영 가능 짧은 타각, 사소한 패턴 이탈처럼 실근로 판단에 영향이 적은 경우입니다.
실근로 여부가 불명확한 이상 충돌 또는 미확정 + 본시간 반영 보류 휴가 시간대 근무, 휴일 장시간 타각, 출장·외근 중복처럼 판단 근거가 더 필요한 경우입니다.
운영상 명백한 모순 오류 + 본시간 반영 불가 휴무일 연차, 중복 기안, 상호 배타적 승인처럼 동시에 성립할 수 없는 경우입니다.

권장 일별 계산 순서

순서 계산 단계 핵심 판단
1캘린더/근로제로 근무일·휴일 판정그날이 원래 어떤 날이었는지 확정
2근무패턴 적용 여부 판단패턴 미사용, 추가 근무, 단축 근무 중 어떤 분기인지 확정
3기준근무시간 계산 및 예상 프레임 생성근로제 시간에 패턴 시간을 가감해 비교 기준을 생성
4원시 타각/외출/휴게 정규화있는 그대로의 사실 데이터 정리
5출퇴근 수정 승인 반영원시값을 교정된 기준값으로 전환
6휴가 승인 반영근무의무구간을 휴가 상태로 대체
7출장/외근/연장 승인 반영부족구간 보완, 근무 시간 기준/신청 시간 기준, 연장 승인 여부 반영
8실제 휴게 우선, 자동 휴게 보조 적용휴게시간 차감 기준 정리
9근무 시간 반영 경로 결정정상, 근태 이상, 연장 자동 인정, 전자결재 인정 중 어느 경로인지 분류
10판정대상시간 산출교정실적과 보완근무를 반영한 후보 시간 계산
11인정근로시간 산출휴게·제외시간 차감 후 일별 최종 시간 계산
12소정/연장/야간/휴일 분해총시간과 가산시간 속성 분리
13이상 상태 생성누락 타각, 승인 미존재, 충돌건 분류
14관리자 보정 및 확정예외 처리, 로그 기록

법령 관점 확인 포인트

  • 휴게는 실제로 자유 이용이 가능한지, 자동 차감만으로 처리해도 되는지 구분해야 합니다.
  • 연장 승인 여부와 실제 연장근로 발생 여부는 같은 개념이 아니므로 분리해 설명해야 합니다.
  • 출장·외근 자동인정은 사업장 밖 간주근로와 유사한 성격을 띠므로 실제 산정 곤란 여부와 통상 필요시간 관점을 함께 봐야 합니다.
  • 휴일근로와 야간근로는 총 인정근로시간을 다시 더하는 개념이 아니라, 이미 인정된 시간의 속성값으로 분해해야 합니다.
  • 중소·중견기업 기준이라면 우선 상시 5인 이상 일반 사업장을 기본 전제로 보되, 선택근로·탄력근로·예외직군은 별도 체크가 필요합니다.
