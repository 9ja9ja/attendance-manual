# 근태매뉴얼 로컬 설정

이 폴더의 HTML 매뉴얼 작성 기준은 아래 문서를 따른다.

- [근태매뉴얼작성규칙.md](./근태매뉴얼작성규칙.md)
- [근태관리_통합매뉴얼_구성계획.md](./근태관리_통합매뉴얼_구성계획.md)

핵심 원칙은 다음과 같다.

1. `index.html + 업무별 상세 HTML` 구조를 유지한다.
2. 기존 글로벌 인덱스에는 바로 연결하지 않는다.
3. 화면 설명만이 아니라 `업무 의미`, `정책 영향`, `마감 영향`까지 함께 설명한다.
4. iframe 기반 인덱스는 로컬 서버로 실행한다.
5. 새 페이지를 추가할 때는 인덱스 링크와 연관 페이지 링크를 함께 갱신한다.
6. 새 전문용어가 등장하면 `용어집.html`에도 함께 추가한다.
7. **작업 시작 전에 반드시 `git pull`을 먼저 실행한다.** 팀원이 원격에서 수정한 내용이 있을 수 있으므로, 최신 상태를 받은 후 작업해야 충돌을 방지할 수 있다.

## Git 협업 규칙

이 매뉴얼은 GitHub(`9ja9ja/attendance-manual`)으로 관리되며, push하면 Vercel(`attendance-manual.vercel.app`)에 자동 배포된다.

- **작업 시작 전**: `git pull` 필수 (최신 상태 동기화)
- **작업 완료 후**: `git add .` → `git commit -m "변경 내용"` → `git push`
- 팀원이 동시에 같은 파일을 수정하면 충돌이 발생할 수 있으므로, 가능하면 작업 범위를 나눠서 진행한다.

실행은 아래 배치 파일을 사용한다.

- [0.근태매뉴얼_보기시작파일.bat](./0.근태매뉴얼_보기시작파일.bat)

## 인덱스 구조 (index.html)

index.html은 **좌측 8개 업무군 + 상단 서브탭** 구조로 구성된다.
좌측 메뉴(nav-item)를 클릭하면 해당 업무군의 서브탭 바가 노출되고, 서브탭을 클릭하면 iframe으로 상세 HTML이 로드된다.

## 파일 수정 시 체크리스트

1. 기존 파일 수정: 해당 파일만 수정하면 됨
2. 새 파일 추가 시:
   - `index.html`의 해당 업무군 subtab-bar에 탭 버튼 추가
   - `index.html`의 해당 sec-panel에 sub-panel + iframe 추가
   - `index.html`의 descriptions 객체에 항목 추가
   - 관련 파일의 "연결 문서" nav-links에 링크 추가
   - `용어집.html`에 신규 전문 용어 추가
   - 이 `CLAUDE.md`의 구조 맵도 갱신
3. 파일 삭제/이동 시:
   - `index.html`에서 해당 탭·패널·descriptions 제거
   - 다른 파일의 nav-links에서 링크 제거/교체
